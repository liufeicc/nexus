# Nexus 文件面板 - 产品需求分析文档

**文档版本**: v2.0
**创建日期**: 2026-04-09
**最后更新**: 2026-04-11
**作者**: 刘飞 (liufei)
**状态**: 原型阶段（核心交互已实现）

> **当前状态**: 大图标网格视图 + 面包屑导航 + 文件查看器（双击打开文本文件）+ 文件状态栏已实现。
> 右键文件操作（复制路径/新建/删除等）、拖拽交换、快照持久化等功能在规划中，尚未实现。

## 1. 需求概述

### 1.1 背景

Nexus 目前只有一种面板类型——终端面板（TerminalPanel），用户可以在多标签会话中运行终端并自由分割布局。但终端面板只解决了"命令行操作"的需求，对于**文件浏览**和**文件操作**场景，用户仍需要切换到系统文件管理器或依赖终端命令（`ls`、`cd`、`cp` 等）。

为了完善 Nexus 的工作台能力，需要新增第二种面板类型——**文件面板**（Bubble File Panel），让用户在终端旁边直接浏览和操作文件。

### 1.2 核心价值

| 价值点 | 描述 |
|--------|------|
| 减少上下文切换 | 不需要在终端和系统文件管理器之间来回切换 |
| 直观文件浏览 | 大图标视图一目了然，快速识别文件和文件夹 |
| 轻量文件操作 | 复制路径、创建文件等常用操作无需敲命令 |
| 文件查看能力 | 双击文件即可在面板内查看文本内容，底部状态栏支持多文件切换 |

### 1.3 设计原则

- **面板类型可扩展**：复用已有的面板扩展机制（`panelType` 设计），本次新增 `file-browser` 类型。
- **最小改动**：尽量复用现有的布局系统、快照系统、状态管理和 IPC 架构。
- **独立生命周期**：文件面板不依赖 PTY 进程，拥有独立的创建/销毁逻辑。
- **文件浏览定位**：采用大图标网格视图 + 面包屑导航，专注于文件浏览和轻量文件操作。

### 2.2 功能详细设计

#### 2.2.1 创建文件面板

- **入口**：工具栏的 `[+ 文件]` 按钮（独立于终端面板按钮）。
- **根目录**：默认使用当前工作目录，无需路径选择器。
- **流程**：
  1. 用户点击 `[+ 文件]`
  2. 系统创建 `panelType: 'file-browser'` 的面板
  3. 以默认根目录初始化文件面板
  4. 添加到布局树并保存快照

> **注意**：原型阶段使用模拟数据，后续接入 Electron IPC 后，根目录可通过 Electron API 动态获取。

#### 2.2.2 大图标网格视图（核心交互）

##### 基础行为

- 文件面板主体区域以**大图标卡片网格**形式展示当前目录下的所有文件和子目录。
- 每个条目显示：类型图标（文件夹 📂 / 文件 📄）+ 文件名。
- 文件夹按字母顺序排在文件前面，各自按名称字母排序。
- 图标卡片尺寸：约 90x90px，横向排列，自动换行。

##### 目录导航

- **双击文件夹图标**：进入该目录，面板内容刷新，面包屑导航同步更新。
- **面包屑导航栏**：位于标题栏下方，显示从根目录到当前目录的完整路径。
  - 每个路径段可点击（当前目录段除外），点击后跳转到对应层级。
  - 路径过长时，中间部分折叠为 `...`，点击可展开。
- **返回上级**：点击面包屑中的上一级路径段，或使用快捷键 `Backspace`。

##### 边界处理

| 场景 | 处理方式 |
|------|----------|
| 空目录 | 网格区域显示"空目录"提示 |
| 权限不足无法读取 | 面板显示"无法访问该目录"提示 |
| 文件数量过多（> 200） | 网格区域支持垂直滚动 |
| 根目录被外部删除 | 面板提示"路径不存在" + "重新选择目录"按钮 |

#### 2.2.3 文件查看器

双击文件即可在文件面板内查看文件内容，采用**全屏替换模式**。

##### 打开文件

- **触发**：在文件网格中双击文件图标。
- **视图切换**：文件网格区域替换为文本查看视图（`<pre>` 标签，保持格式）。
- **标题栏**：显示 `📁 文件浏览器 - 文件名`，右侧有关闭按钮。
- **状态栏同步**：该文件自动添加到文件状态栏并高亮。

##### 切换文件

- 点击文件状态栏中的其他文件标签，查看内容跟随切换。
- 高亮标签跟随当前查看的文件。

##### 关闭文件

- 点击文件查看器标题栏的 `×` 按钮，或状态栏中当前文件的 `×` 按钮。
- 文本查看区隐藏，恢复文件网格显示。
- 如果还有其他已打开文件，切换到第一个；否则回到网格浏览状态。

##### 文本查看视图结构

```
┌──────────────────────────────────────────┐
│  📁 文件浏览器 - myproject          [×]  │  ← 标题栏
├──────────────────────────────────────────┤
│  📁 root > src > index.ts                │  ← 面包屑
├──────────────────────────────────────────┤
│                                          │
│  1  import { createApp } from 'vue';     │
│  2  import App from './App.vue';         │  ← 文本查看区
│  3                                       │     <pre> 标签
│  4  const app = createApp(App);          │
│  ...                                     │
│                                          │
├──────────────────────────────────────────┤
│  [📄 index.ts ×]                         │  ← 文件状态栏
└──────────────────────────────────────────┘
```

> **当前阶段**：仅支持文本文件查看。后续可扩展支持 Markdown 渲染、代码高亮等。

#### 2.2.4 文件状态栏

位于文件面板最底部，固定 28px 高度，显示已打开的文件标签。

##### 结构

```
┌──────────────────────────────────────────────────────────────────┐
│  [📄 index.ts]  [📄 README.md ×]  [📄 config.json ×]            │
│   ↑当前查看      ↑其他文件      ↑过长时截断... ×                  │
└──────────────────────────────────────────────────────────────────┘
```

##### 行为

| 元素 | 功能 |
|------|------|
| 文件标签 | 显示文件名和类型图标，点击可切换查看该文件 |
| 当前高亮 | 正在查看的文件以强调色背景高亮 |
| 关闭按钮 | 悬停时显示 `×`，点击关闭该文件 |
| 空状态 | 未打开文件时显示"未打开文件"提示 |
| 文件过多 | 支持横向滚动 |

#### 2.2.5 文件/文件夹条目展示

在文件面板的主内容区域（网格状态），以图标卡片形式展示当前目录下的所有条目：

- **卡片结构**：
  - 中央：类型图标（文件夹 📂 / 文件 📄 / 符号链接 🔗）
  - 下方：名称（超长时截断，鼠标悬停显示完整名称 tooltip）

- **排序**：文件夹在前，文件在后；各自按名称字母排序。
- **隐藏文件**：默认显示隐藏文件（以 `.` 开头的文件）。

#### 2.2.6 复制文件路径（文本层面）

- **方式**：右键点击条目时，上下文菜单中包含"复制路径"选项。
- **反馈**：复制成功后显示 Toast 提示"已复制路径"。

#### 2.2.7 文件复制/粘贴（操作层面）

##### 复制文件

- 右键点击文件/文件夹条目，上下文菜单中包含"复制文件"选项。
- 点击后，将该文件/文件夹路径记录到**内部剪贴板**（Zustand store 中维护）。
- 反馈：Toast 提示"已复制：xxx"。

##### 粘贴文件

- 当内部剪贴板中有内容时，在文件面板的空白区域右键，上下文菜单中显示"粘贴"选项（可用状态）。
- 点击后，将复制的文件/文件夹复制到当前目录。
- **同名冲突处理**：如果目标目录已存在同名文件/文件夹，弹出确认对话框：
  - "跳过"：不覆盖
  - "替换"：覆盖原文件
  - "保留两者"：自动重命名为 `xxx_copy` 或 `xxx (1)`
- 反馈：Toast 提示"已粘贴"或操作结果。

##### 内部剪贴板 vs 系统剪贴板

| 剪贴板类型 | 用途 | 存储位置 |
|-----------|------|---------|
| 系统剪贴板 | 复制文件路径文本，可在应用外粘贴 | Electron `clipboard` API |
| 内部剪贴板 | 文件操作层面的复制/粘贴 | Zustand store（`fileClipboard` 状态） |

#### 2.2.8 创建空文件

- 在文件面板的空白区域右键，上下文菜单中包含"新建文件"选项。
- 点击后，弹出输入框让用户输入文件名（默认值为 `untitled.txt`）。
- 确认后，在当前目录创建空文件。
- 如果文件名冲突，提示用户重新输入。
- 反馈：Toast 提示"文件已创建"。

#### 2.2.9 右键上下文菜单

##### 文件/文件夹条目右键菜单

#### 2.2.10 快照持久化

- 关闭会话后重新打开，文件面板应恢复：
  - 根目录路径
  - 当前浏览的目录路径
  - 面板在布局中的位置
  - 面板大小比例
- **不保存**：文件查看器的打开状态（关闭面板时自动关闭所有已打开文件）。

## 3. 交互设计详细说明

### 3.1 文件浏览完整交互流程

```
用户创建文件面板（点击 [+ 文件] 按钮）
  ↓
文件面板以大图标网格显示根目录内容
  ↓
┌──────────────────────────────────────┐
│  📁 文件浏览器                  [×] │  ← 标题栏
├──────────────────────────────────────┤
│  📁 root > src > components          │  ← 面包屑
├──────────────────────────────────────┤
│ ┌────┐ ┌────┐ ┌────┐ ┌────┐         │
│ │ 📂 │ │ 📂 │ │ 📄 │ │ 📄 │         │  ← 文件网格
│ └────┘ └────┘ └────┘ └────┘         │
└──────────────────────────────────────┘

--- 双击进入目录 ---
用户双击文件夹图标
  ↓
面板内容刷新，显示该文件夹内容
  ↓
面包屑导航更新：root > src > components > 目标文件夹

--- 面包屑跳转 ---
用户点击面包屑中的某一级路径
  ↓
面板内容跳转到该层级目录
  ↓
面包屑同步更新

--- 双击打开文件 ---
用户双击文件图标
  ↓
文件网格区域替换为文本查看视图
  ↓
文件添加到状态栏并高亮

--- 状态栏切换 ---
用户点击状态栏中的其他文件标签
  ↓
文本查看区刷新为该文件内容
  ↓
高亮标签跟随移动

--- 关闭文件 ---
用户点击文件查看器标题栏的 × 或状态栏的 ×
  ↓
文本查看区隐藏，恢复文件网格
```

### 3.2 动画参数

| 动画 | 持续时间 | 缓动函数 | 效果 |
|------|---------|---------|------|
| 文件网格 → 查看器切换 | 200ms | ease-out | display 切换，无过渡动画 |
| Toast 提示 | 300ms 出现，300ms 消失 | ease-out/ease-in | 透明度 + translateY |
| 状态栏标签 hover | 150ms | ease | 背景色过渡 |

### 3.3 键盘交互

| 快捷键 | 行为 |
|--------|------|
| `Enter` | 确认根目录选择（在路径选择器中） |
| `Escape` | 退出新建文件输入框；关闭模态窗口 |
| `Backspace` | 返回上级目录 |
| `双击` | 进入目录 / 打开文件 |

## 4. 技术实现要点

### 4.1 需要新增的 IPC 接口

以下接口需要在主进程（`ipc-handlers.ts`）中注册，并在预加载脚本（`preload.ts`）中暴露给渲染进程。

| 接口名 | 方向 | 参数 | 返回值 | 描述 |
|--------|------|------|--------|------|
| `file:list` | 渲染→主 | `{ dirPath: string }` | `{ entries: FileEntry[], error?: string }` | 列出目录下所有文件和子目录 |
| `file:copy` | 渲染→主 | `{ sourcePath: string, destPath: string, overwrite?: boolean }` | `{ success: boolean, error?: string }` | 复制文件或目录 |
| `file:createEmpty` | 渲染→主 | `{ dirPath: string, fileName: string }` | `{ success: boolean, error?: string }` | 在指定目录创建空文件 |
| `file:exists` | 渲染→主 | `{ path: string }` | `{ exists: boolean, isDirectory: boolean }` | 检查路径是否存在及类型 |

> **注**：`file:list` 的 `FileEntry` 结构：
> ```typescript
> interface FileEntry {
>   name: string
>   isDirectory: boolean
>   size?: number
>   modifiedAt?: number
>   path: string  // 完整路径
> }
> ```

### 4.2 IPC 频道定义

在 `src/core/constants/ipc-channels.ts` 中新增：

```typescript
// 文件操作
FILE_LIST: 'file:list',
FILE_COPY: 'file:copy',
FILE_CREATE_EMPTY: 'file:createEmpty',
FILE_EXISTS: 'file:exists',
```

### 4.3 状态管理扩展

#### 4.3.1 PanelState 扩展

需要扩展 `PanelState` 以支持文件面板：

```typescript
export type PanelType = 'terminal' | 'browser' | 'file-browser'

export interface PanelState {
  id: string
  panelType: PanelType       // 新增，默认 'terminal'
  ptyId?: string             // 改为可选
  title: string
  cwd?: string               // 改为可选，终端面板使用
  url?: string               // 浏览器面板使用
  rootPath?: string          // 文件面板：根目录路径
  currentPath?: string       // 文件面板：当前浏览的目录路径
}
```

> **注意**：这是一个渐进式改造。如果浏览器面板的需求已实现 `panelType`，则直接在其基础上增加 `'file-browser'` 类型。如果浏览器面板尚未实现，建议先统一引入 `panelType` 机制。

#### 4.3.2 新增 Zustand 状态

```typescript
interface AppState {
  // ... 现有状态

  // 文件面板内部剪贴板
  fileClipboard: {
    sourcePath: string
    isDirectory: boolean
    name: string
  } | null

  // 文件面板动作
  createFilePanel: (rootPath: string) => Promise<string>  // 创建文件面板
  setFileClipboard: (data: { sourcePath: string; isDirectory: boolean; name: string } | null) => void
  clearFileClipboard: () => void
}
```

#### 4.3.3 需要修改的现有方法

| 方法 | 修改内容 |
|------|----------|
| `closePanel` | 判断 `panelType`：`terminal` 需 kill PTY，`file-browser` 直接移除 |
| `restorePanelsFromData` | 根据 `panelType` 分别处理，文件面板不需要重建 PTY |
| `saveSnapshot` | 快照数据增加 `panelType`、`rootPath`、`currentPath` 字段 |

### 4.4 组件结构

#### 4.4.1 新增组件

#### 4.4.2 组件职责

**FileBrowserPanel.tsx**
- 文件面板的根组件
- 管理根目录、当前目录状态
- 控制文件网格与文件查看器的显示切换
- 维护已打开文件列表（`openFilesMap`）
- 处理面板标题更新

**FileGrid.tsx**

### 4.6 主进程实现要点

#### 4.6.1 文件列表 IPC 处理器

```typescript
ipcMain.handle(IPC_CHANNELS.FILE_LIST, async (_, dirPath: string) => {
  try {
    const entries = await fs.promises.readdir(dirPath, { withFileTypes: true })
    const fileEntries = entries.map(entry => {
      const fullPath = path.join(dirPath, entry.name)
      return {
        name: entry.name,
        isDirectory: entry.isDirectory(),
        path: fullPath,
      }
    })
    // 文件夹在前，文件在后，各自按名称排序
    fileEntries.sort((a, b) => {
      if (a.isDirectory !== b.isDirectory) return b.isDirectory ? 1 : -1
      return a.name.localeCompare(b.name)
    })
    return { entries: fileEntries }
  } catch (error) {
    return { entries: [], error: (error as Error).message }
  }
})
```

#### 4.6.2 文件复制 IPC 处理器

```typescript
ipcMain.handle(IPC_CHANNELS.FILE_COPY, async (_, params: {
  sourcePath: string
  destPath: string
  overwrite?: boolean
}) => {
  try {
    const { sourcePath, destPath, overwrite = false } = params
    const stat = await fs.promises.stat(sourcePath)

    if (stat.isDirectory()) {
      await copyDirectory(sourcePath, destPath, overwrite)
    } else {
      if (!overwrite && await fs.promises.access(destPath).then(() => true).catch(() => false)) {
        return { success: false, error: '目标文件已存在' }
      }
      await fs.promises.copyFile(sourcePath, destPath)
    }
    return { success: true }
  } catch (error) {
    return { success: false, error: (error as Error).message }
  }
})

// 递归复制目录
async function copyDirectory(src: string, dest: string, overwrite: boolean): Promise<void> {
  await fs.promises.mkdir(dest, { recursive: true })
  const entries = await fs.promises.readdir(src, { withFileTypes: true })
  for (const entry of entries) {
    const srcPath = path.join(src, entry.name)
    const destPath = path.join(dest, entry.name)
    if (entry.isDirectory()) {
      await copyDirectory(srcPath, destPath, overwrite)
    } else {
      if (!overwrite && await fs.promises.access(destPath).then(() => true).catch(() => false)) {
        continue // 跳过已存在的文件
      }
      await fs.promises.copyFile(srcPath, destPath)
    }
  }
}
```

#### 4.6.3 创建空文件 IPC 处理器

```typescript
ipcMain.handle(IPC_CHANNELS.FILE_CREATE_EMPTY, async (_, params: {
  dirPath: string
  fileName: string
}) => {
  try {
    const fullPath = path.join(params.dirPath, params.fileName)
    const exists = await fs.promises.access(fullPath).then(() => true).catch(() => false)
    if (exists) {
      return { success: false, error: '文件已存在' }
    }
    await fs.promises.writeFile(fullPath, '')
    return { success: true }
  } catch (error) {
    return { success: false, error: (error as Error).message }
  }
})
```

## 5. 边界情况和异常处理

### 5.1 权限问题

| 场景 | 处理方式 |
|------|----------|
| 根目录无读权限 | 面板显示错误提示"无法访问该目录"，提供"重新选择目录"按钮 |
| 子目录无读权限 | 气泡中显示"无法访问"，不提供进一步操作 |
| 粘贴到无写权限目录 | Toast 提示"没有写入权限"，粘贴操作失败 |
| 创建文件到无写权限目录 | Toast 提示"没有写入权限" |

### 5.2 大目录性能

| 场景 | 处理方式 |
|------|----------|
| 目录下文件数 > 200 | 网格区域支持垂直滚动 |
| 目录下文件数 > 500 | 建议改用终端命令（如 `ls`）浏览 |
| 快速双击文件夹 | 正常处理，无额外性能影响 |

### 5.3 路径相关

| 场景 | 处理方式 |
|------|----------|
| 根目录被外部删除 | 面板检测到路径不存在，显示提示并提供"重新选择目录" |
| 文件名包含特殊字符 | 正常显示，复制路径时使用完整路径字符串 |
| Windows 路径 vs Linux 路径 | 使用 Node.js `path` 模块自动处理路径分隔符 |
| 符号链接（symlink） | 显示为带箭头的文件夹图标，点击/悬停时跟随符号链接目标 |

### 5.4 并发冲突

| 场景 | 处理方式 |
|------|----------|
| 粘贴时目标文件被其他进程占用 | IPC 返回错误，Toast 提示"文件被占用，请稍后重试" |
| 复制过程中源文件被删除 | 捕获异常，Toast 提示"源文件不存在，复制已取消" |
| 创建文件时同名文件突然出现 | IPC 返回错误，提示用户重新输入文件名 |
| 查看文件时文件被外部修改 | 文本查看区不自动刷新，用户需关闭重新打开 |

### 5.5 文件查看器边界

| 场景 | 处理方式 |
|------|----------|
| 查看非文本文件（图片、二进制） | 提示"不支持的文件类型"，保持网格显示 |
| 超大文本文件（> 1MB） | 提示"文件过大，建议在终端中使用 cat/less 查看" |
| 文件编码非 UTF-8 | 尝试自动检测，失败时提示编码问题 |
| 查看过程中文件被外部删除 | 不影响当前显示，关闭时提示文件不存在 |

## 8. 风险与约束

### 8.1 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 大量文件导致网格渲染性能下降 | 卡顿 | 低 | 文件数超过 200 时启用滚动，超过 500 提示使用终端命令 |
| `panelType` 字段引入导致现有代码不兼容 | 构建失败或运行时错误 | 低 | 所有现有代码中引用 `ptyId` 和 `cwd` 的地方加可选链或空值检查 |
| 文件复制大目录耗时长 | 用户等待 | 中 | 大目录复制时显示进度提示，可取消 |
| 大文本文件查看导致内存占用过高 | 性能下降 | 低 | 限制查看文件大小（> 1MB 提示使用终端） |

### 8.2 约束

| 约束 | 描述 |
|------|------|
| 不依赖 PTY | 文件面板的生命周期独立于 PTY 进程 |
| 复用现有架构 | 布局系统、快照系统、主题系统不做结构性改动 |
| Electron 30 | 使用当前项目已使用的 Electron 版本，不升级 |
| 仅 Linux 环境测试 | 开发环境为 Linux，需要在 Windows/macOS 上额外测试 |
