## 1. 系统概述

### 1.1 产品介绍

**Nexus** (Terminal Views) 是一款基于 Electron 开发的图形化终端复用工具，提供类似 tmux 的多路复用功能，但通过直观的 GUI 界面降低使用门槛。

### 1.2 目标用户

- 觉得 tmux 快捷键复杂难记的开发者
- 需要频繁管理多个终端会话的前端/全栈工程师
- 希望可视化操作终端面板的 DevOps 人员
- 对 terminal workflow 有需求但不愿学习陡峭曲线的初级开发者

### 1.3 核心特性

| 特性 | 说明 |
|------|------|
| 多标签会话 | 支持创建多个独立的终端会话 |
| 分屏功能 | 支持水平/垂直分割终端面板 |
| 多主题切换 | 6 种预设主题配色方案 |
| 图形化操作 | 鼠标点击 + 工具栏按钮操作 |
| 会话管理 | 会话列表、切换、新建、关闭 |

### 1.4 技术规格

- **终端渲染**: xterm.js v5.3.0
- **框架**: Electron + React (设计文档)
- **PTY 支持**: node-pty
- **构建工具**: Vite

### 2.2 区域说明

#### 2.2.1 顶部标题栏

位于窗口最上方，采用**自定义无框窗口**设计（隐藏系统原生标题栏），包含：

| 元素 | 功能 |
|------|------|
| Nexus Logo | 应用标识 |
| 主题切换按钮 | 点击打开主题选择下拉菜单 |
| 帮助按钮 | 占位按钮，暂时无功能 |
| 最小化按钮 | 点击将应用最小化到任务栏 |
| 最大化/还原按钮 | 点击切换最大化/还原状态，图标随之变化 |
| 关闭窗口按钮 | 点击弹出确认对话框，确认后关闭应用 |

**窗口控制按钮详细说明**:

```
[主题图标] [？] [-] [□/▢] [×]
  主题切换  帮助  最小化 最大化  关闭
```

| 按钮 | 图标 | 功能 |
|------|------|------|
| 最小化 | 横线 (—) | 将窗口最小化到任务栏 |
| 最大化 | 方形 (□) | 窗口最大化，充满整个屏幕 |
| 还原 | 重叠方形 (▢) | 从最大化状态恢复到之前的大小 |
| 关闭 | 叉号 (×) | 弹出"确认关闭 Nexus 吗？"对话框 |

**窗口规格**:
- 初始大小：1200×800 像素
- 最小大小：800×600 像素

#### 2.2.2 左侧会话列表

位于窗口左侧，宽度可调（默认 224px），包含：

| 元素 | 功能 |
|------|------|
| "会话"标题 | 侧边栏标识，带新建按钮 |
| 最近使用组 | 显示最近使用的会话（最多 3 个），可展开/收起 |
| 全部会话组 | 显示所有可用会话 |
| 状态指示器 | 绿色圆点表示活动会话，灰色表示非活动 |
| 新建会话按钮 | 位于底部，点击创建新会话 |

**侧边栏宽度调整**:
- 侧边栏与主区域之间有可拖拽分割线
- 鼠标悬停时显示拖拽手柄和收起按钮
- 支持拖拽调整宽度（160px - 400px）
- 支持点击按钮收起/展开侧边栏

**最近使用会话**:
- 显示最近使用的 3 个会话
- 点击"最近使用"标签或箭头可展开/收起
- 展开/收起状态自动保存到数据库
- 活动会话排在首位，其余按 `last_used_at` 降序排列

#### 2.2.3 顶部工具栏

位于主区域上方，高度 36px (h-9)，包含：

| 按钮组 | 按钮 | 功能 |
|--------|------|------|
| 分屏模式 | ⊞ | 选择水平分屏模式（新面板出现在右侧） |
| 分屏模式 | ⊟ | 选择垂直分屏模式（新面板出现在下方） |
| 面板操作 | + | 新增面板（弹出路径选择对话框） |
| 面板操作 | × | 关闭当前选中的面板 |

**分屏按钮说明**:

两个分屏按钮同时显示在工具栏上，点击即可选择分屏方向：

| 模式 | 说明 | 效果示意 |
|------|------|---------|
| 水平分屏 | 新增面板时横向排列 | `[面板 1][面板 2][面板 3]` |
| 垂直分屏 | 新增面板时纵向排列 | `[[面板 1]][[面板 2]][[面板 3]]` |

**按钮状态指示**:
- 当前激活的分屏模式会以强调色高亮显示
- 默认模式为水平分屏

**分屏按钮逻辑**:
- **有选中面板时**：点击"+"会按当前分屏模式，在选中面板旁边创建新面板（树形嵌套分割）
- **没有面板时**：点击"+"会创建第一个面板，占满整个容器

**分屏效果示意**:

水平分屏模式（横向排列）：
```
1 个面板           2 个面板           3 个面板
┌────────┐        ┌────┬────┐       ┌───┬───┬───┐
│        │        │    │    │       │   │   │   │
│  P1    │   →    │ P1 │ P2 │  →    │P1 │P2 │P3 │
│        │        │    │    │       │   │   │   │
└────────┘        └────┴────┘       └───┴───┴───┘
100% 宽度          各 50% 宽度        各 33% 宽度
```

垂直分屏模式（纵向排列）：
```
1 个面板           2 个面板           3 个面板
┌────────┐        ┌────────┐        ┌────────┐
│        │        │  P1    │        │  P1    │
│  P1    │   →    ├────────┤   →    ├────────┤
│        │        │  P2    │        │  P2    │
└────────┘        └────────┘        ├────────┤
100% 高度          各 50% 高度        │  P3    │
                                    └────────┘
                                    各 33% 高度
```

#### 2.2.4 终端面板区域

位于窗口中央，占据主要空间，支持：

- **网格布局**: 树形嵌套布局，支持任意嵌套分屏
- **多面板**: 每个面板独立运行终端实例
- **可调整**: 面板间的分割线可拖拽调整大小
- **自动适应**: 终端内容自动适应面板尺寸

**单个面板结构**:
```
┌─────────────────────────────────────┐
│ [状态] localhost  │  bash - ~  │ × │  ← 面板标题栏
├─────────────────────────────────────┤
│                                     │
│           终端内容区域               │
│           (xterm.js 渲染)            │
│                                     │
└─────────────────────────────────────┘
```

#### 2.2.5 底部状态栏

位于窗口底部，高度 24px (h-6)，显示：

| 信息 | 说明 | 图标 |
|------|------|------|
| 活动会话 | 当前活跃的会话名称 | 绿色状态点 |
| 当前路径 | 工作目录路径 | 文件夹图标 |
| Shell 信息 | Shell 类型和版本 | 终端图标 |
| 终端尺寸 | 行列数 (如 80x24) | 网格图标 |
| Git 状态 | 当前分支状态 | Git 图标 |
| 状态图标 | 收藏、通知等快捷操作 | 心形/铃铛图标 |

#### 2.2.6 侧边栏调整功能

侧边栏与主区域之间设有可拖拽分割线，支持调整宽度和收起/展开操作。

**分割线外观**:
- 默认状态：透明，宽度 6px
- 悬停状态：显示半透明背景，中央出现垂直线条
- 拖拽状态：背景高亮，禁止过渡动画

**收起/展开按钮**:
- 位置：分割线中央
- 显示条件：鼠标悬停在分割线上时
- 按钮样式：深色半透明背景，带毛玻璃效果
- 箭头图标：指示当前状态（→展开，←收起）

**调整范围**:
- 最小宽度：160px
- 最大宽度：400px
- 默认宽度：224px

### 2.3 侧边栏调整操作

#### 2.3.1 调整侧边栏宽度

**操作步骤**:
```
1. 将鼠标移到左侧会话列表与右侧面板之间的分割线上
2. 当鼠标变为左右箭头形状（↔）时，按住鼠标左键
3. 左右拖动调整侧边栏宽度
   - 向右拖动：增加侧边栏宽度
   - 向左拖动：减小侧边栏宽度
4. 释放鼠标完成调整
```

**注意事项**:
- 拖拽时侧边栏禁止过渡动画，确保跟手流畅
- 宽度限制在 160px-400px 范围内
- 调整后宽度自动保存，下次打开自动恢复

#### 2.3.2 收起/展开侧边栏

**操作步骤**:
```
1. 将鼠标移到分割线上
   - 分割线会高亮显示（半透明背景）
   - 分割线中央出现按钮
2. 点击中央的按钮
   - 点击后侧边栏收起或展开
   - 按钮箭头方向随之改变
```

**箭头方向说明**:
| 箭头方向 | 状态 | 操作结果 |
|---------|------|---------|
| → (向右) | 展开 | 点击后收起侧边栏 |
| ← (向左) | 收起 | 点击后展开侧边栏 |

**使用场景**:
| 场景 | 推荐状态 | 说明 |
|------|---------|------|
| 需要更大终端面板区域 | 收起 | 最大化代码/终端可视区域 |
| 频繁切换多个会话 | 展开 | 方便快速查看和切换 |
| 专注单个会话工作 | 收起 | 减少视觉干扰 |
| 多会话并行工作 | 展开 | 便于监控各会话状态 |

#### 2.3.3 状态持久化

侧边栏的状态会自动保存到数据库：
- **宽度设置**: 保存到 SQLite 数据库（配置表）
- **收起/展开状态**: 保存到 SQLite 数据库
- **恢复时机**: 应用启动时自动加载上次保存的设置

**终端欢迎信息**:
```
新面板创建时会显示欢迎信息：

 欢迎使用 Nexus
Terminal Version: 80x24
Shell: bash 5.1.8
Working Directory: /home/user/projects

提示：这是一个模拟的终端，可用于输入命令
输入 clear 清屏，输入 help 查看帮助

说明：
- "欢迎使用 Nexus" 以绿色显示
- "提示：" 以黄色显示
- 工作目录路径与实际创建路径一致
```

### 2.4 窗口控制按钮

Nexus 使用自定义无框窗口（隐藏系统原生标题栏），在顶部标题栏右侧提供窗口控制按钮。

#### 2.4.1 最小化

```
操作：点击标题栏右侧的横线按钮 (—)
效果：窗口最小化到任务栏，应用继续运行
```

#### 2.4.2 最大化/还原

```
操作：点击标题栏右侧的方形按钮
效果：
- 当前为窗口状态时 → 最大化，充满整个屏幕（图标变为重叠方形）
- 当前为最大化状态时 → 还原为之前的大小（图标变为单方形）

提示：最大化/还原状态实时同步，按钮图标随之切换。
```

#### 2.4.3 关闭应用

```
操作：点击标题栏右侧的叉号按钮 (×)
效果：弹出确认对话框 "确定要关闭 Nexus 吗？"
```

**关闭确认对话框**:
```
┌─────────────────────────────────┐
│  ⚠ 确认关闭                     │
├─────────────────────────────────┤
│  确定要关闭 Nexus 吗？           │
├─────────────────────────────────┤
│              [取消]  [确定]      │
└─────────────────────────────────┘
```

- 点击"取消"：对话框关闭，应用继续运行
- 点击"确定"：保存窗口状态（位置、大小、最大化状态）后关闭应用

**窗口规格**:
- 初始大小：1200×800 像素
- 最小大小：800×600 像素

## 3. 功能操作详解

### 3.1 会话管理

#### 3.1.1 新建会话

**会话列表位置**: 位于窗口**左侧边栏**（宽度可调，默认 224px），占据左半边垂直区域。

```
窗口布局示意:
┌──────────────────────────────────────────┐
│              顶部标题栏                   │
├──────────┬───────────────────────────────┤
│          │                               │
│ 会话     │                               │
│ 列表     │      主区域 (终端面板)          │
│ (左侧)   │                               │
│          │                               │
│ [+新建]  │                               │
│  ↑       │                               │
│  点击这里 │                               │
└──────────┴───────────────────────────────┘
```

**方法一**: 点击侧边栏底部按钮
```
操作步骤:
1. 看向窗口左侧边栏
2. 找到侧边栏最底部的"[+] 新建会话"按钮
3. 点击该按钮
结果：创建新会话并自动切换到该会话，主区域显示新会话的终端面板
```

**方法二**: 使用右键菜单
```
操作：在终端面板空白处右键点击 → 选择"新建会话"
结果：创建新会话并自动切换到该会话
```

**方法三**: 点击侧边栏标题栏加号按钮
```
操作步骤:
1. 看向窗口左侧边栏
2. 找到侧边栏顶部的"会话"标题栏
3. 点击标题栏右侧的"+"按钮
结果：创建新会话并自动切换到该会话
```

**提示**: 当所有会话都被删除后，点击新建会话按钮会创建新会话并自动切换过去，主区域会显示该会话的终端面板。

#### 3.1.2 切换会话

```
操作步骤：
1. 在左侧会话列表中查看可用会话
2. 点击目标会话项（如 new-shell）
3. 活动会话显示绿色状态指示器
4. 主区域切换到对应会话的终端面板
```

**会话状态说明**:

| 状态 | 显示 | 说明 |
|------|------|------|
| 活动 | 绿色圆点 + 高亮背景 | 当前正在使用的会话 |
| 非活动 | 灰色圆点 + 普通背景 | 后台运行的会话 |

#### 3.1.3 关闭会话

关闭会话会终止该会话下的所有终端进程。

**方法一：点击会话项右侧的 × 按钮**
```
操作步骤：
1. 将鼠标悬停在要关闭的会话项上
2. 点击右侧出现的 × 按钮
3. 在确认对话框中点击"确定"
```

**方法二：右键菜单**
```
操作步骤：
1. 在会话项上点击鼠标右键
2. 在弹出的菜单中选择关闭选项
```

**方法三：关闭面板**
```
操作步骤：
1. 在工具栏点击关闭面板按钮
2. 或在面板标题栏点击关闭按钮
3. 或在终端面板上右键 → 选择"关闭面板"
```

**注意**:
- 关闭会话会终止该会话下的所有终端进程
- 关闭操作不可撤销，请确保已保存重要数据

#### 3.1.4 重命名会话

**方法一**: 点击编辑按钮（铅笔图标）
```
操作步骤:
1. 将鼠标悬停在要重命名的会话项上
2. 点击右侧出现的铅笔图标
3. 在弹出的模态窗口中输入新名称
4. 点击"确定"或按回车键保存

注意：选中的会话（活动会话）的编辑按钮始终可见，
```

### 3.1.7 新增面板

Nexus 采用**一对多面板模式**：一个会话可以包含多个终端面板，通过布局模式控制排列方式。

**核心概念**:
- **布局模式**: 通过顶部"布局"按钮选择（左右分屏/上下分屏）

```
操作步骤:

4. 选择常用路径或输入自定义路径
5. 点击"确定"
6. 创建新面板，占满整个容器区域

状态变化:
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  空状态提示  │ →  │  选择路径    │ →  │  新面板 P1   │
│              │    │  (对话框)    │    │  (100% 容器) │
└──────────────┘    └──────────────┘    └──────────────┘
```

**场景二：已有 1 个面板，新增第 2 个面板**

```
操作步骤:

   - 左右分屏：两个面板横向排列
   - 上下分屏：两个面板纵向排列
2. 点击"+"按钮
3. 选择路径后创建新面板

效果示意（左右分屏模式）:
┌──────────┐         ┌──────┬──────┐
│          │         │      │      │
│   P1     │   →    │  P1  │  P2  │
│          │         │      │      │

100% 宽度            各 50% 宽度
```

**场景三：已有多个面板，继续新增**

```
效果示意（左右分屏模式）:
┌──────┬──────┐         ┌───┬───┬───┐
```

- 新增面板时会创建新的终端实例，独立运行
- 每个面板有独立的工作目录和命令历史
- 关闭面板不会影响同一会话的其他面板

### 3.2 树形嵌套布局

Nexus 采用**树形嵌套布局**，支持灵活的分屏操作。

#### 3.2.1 什么是树形嵌套布局？

**核心理念**: 每次分割只针对**当前选中的面板**，而不是所有面板。

**布局模式**:
| 模式 | 分割对象 | 说明 |
|------|---------|------|
| 水平分屏 | 选中的面板 | 新面板出现在选中面板的右侧 |
| 垂直分屏 | 选中的面板 | 新面板出现在选中面板的下方 |

**效果示意**:

**场景 1**: 先左右分屏，再在右侧面板上下分屏
```
步骤 1: 创建面板 1          步骤 2: 左右分屏，创建面板 2
┌─────────────┐            ┌──────────┬──────────┐
│             │            │          │          │
│   面板 1     │     →      │  面板 1   │  面板 2   │
│             │            │          │          │
└─────────────┘            └──────────┴──────────┘

步骤 3: 选中面板 2，上下分屏创建面板 3
┌──────────┬──────────┐
│          │  面板 2   │
│  面板 1   ├──────────┤
│          │  面板 3   │
└──────────┴──────────┘
```

**场景 2**: 先上下分屏，再在下方面板左右分屏
```
步骤 1: 创建面板 1          步骤 2: 上下分屏，创建面板 2
┌─────────────┐            ┌─────────────┐
│             │            │   面板 1    │
│   面板 1     │     →      ├─────────────┤
│             │            │   面板 2    │
└─────────────┘            └─────────────┘

步骤 3: 选中面板 2，左右分屏创建面板 3
┌─────────────────────┐
│      面板 1         │
├──────────┬──────────┤
│  面板 2   │  面板 3   │
└──────────┴──────────┘
```

#### 3.2.2 操作步骤

**分割面板**:
```
1. 点击要分割的面板（该面板高亮显示）
2. 选择分屏模式：
   - 点击工具栏的水平分屏按钮 → 新面板出现在右侧
   - 点击工具栏的垂直分屏按钮 → 新面板出现在下方
3. 点击工具栏的"+"按钮
4. 选择起始路径后创建新面板
```

**关闭面板**:
```
1. 点击面板标题栏的"×"按钮
2. 或在工具栏点击"关闭面板"
3. 或在面板上右键 → 选择"关闭面板"

注意:
- 关闭面板后，如果容器只剩一个面板，会自动展开
- 例如：关闭上图的面板 2 后，布局变为 面板 1 | 面板 3
```

#### 3.2.3 视觉反馈

**选中面板高亮**:
- 活动面板：标题栏高亮显示，可接收键盘输入
- 非活动面板：普通显示，点击后变为活动面板

#### 3.2.4 面板分割线拖拽调整大小

在树形嵌套布局中，相邻面板之间设有可拖拽的分割线，允许用户自定义各面板的相对大小。

**分割线外观**:
```
水平布局（左右排列）:     垂直布局（上下排列）:
┌──────│──────┐          ┌─────────────┐
│      │      │          │   面板 1    │
│  P1  │  P2  │          ├─────────────┤  ← 4px 分割线
│      │      │          │   面板 2    │
└──────│──────┘          └─────────────┘
  ↑ 4px 分割线
```

| 状态 | 外观 | 说明 |
|------|------|------|
| 默认 | 半透明细线 | 几乎不可见，不干扰视觉 |
| 悬停 | 半透明背景 + 明显线条 | 提示可拖拽 |
| 拖拽中 | 高亮背景 + 实线 | 无过渡动画，确保跟手 |

**操作步骤**:
```
1. 将鼠标移到面板间的分割线上
   - 水平布局：鼠标变为左右箭头 (↔)
   - 垂直布局：鼠标变为上下箭头 (↕)
2. 按住鼠标左键不放
3. 拖动分割线调整相邻面板的相对大小
   - 向右/下拖动：左侧/上方面板变大，右侧/下方面板变小
   - 向左/上拖动：左侧/上方面板变小，右侧/下方面板变大
4. 释放鼠标完成调整

注意：面板最小比例限制为约 10%，不能缩为 0。
```

**比例自动保存**:
- 拖动调整结束后，新的面板比例会自动保存到会话快照
- 切换会话或重新打开应用时，面板比例自动恢复

#### 3.3.3 拖动面板交换位置（v2.0）

**功能描述**: 按住面板标题栏拖动到另一个面板位置，释放后两个面板交换位置。

**操作步骤**:

**视觉效果**:
| 状态 | 效果 | 说明 |
|------|------|------|
| 拖动中 | 半透明虚影 + 原面板半透明 | 清楚识别被拖动的面板 |
| 拖入目标 | 目标面板绿色边框 | 提示交换目标位置 |

**拖动效果示意**:
```
拖动前:                  拖动中:                  交换后:
┌──────┬──────┐         ┌──────┬──────┐         ┌──────┬──────┐
│      │      │         │      │      │         │      │      │
│  P1  │  P2  │   →    │  P1  │ ~~~~ │   →    │  P2  │  P1  │
│      │      │         │      │ ~~~~ │         │      │      │
└──────┴──────┘         └──────┴──────┘         └──────┴──────┘
```

**注意事项**:
- 只有一个面板时，不可拖动
- 必须按住面板标题栏（蓝色背景区域）才能拖动

**使用场景**:
| 场景 | 说明 |
|------|------|
| 调整面板布局 | 快速交换面板位置，无需重新分屏 |

### 3.5 终端复制与粘贴

Nexus 支持终端文本的复制与粘贴操作，可通过右键菜单或快捷键完成。

#### 3.5.1 复制终端文本

```
方法一：快捷键 Ctrl+C（智能模式）
1. 在终端中用鼠标拖动选中一段文本
2. 按下 Ctrl+C
3. 屏幕底部显示"已复制到剪贴板"提示
4. 选中的文本已复制到剪贴板，终端选区已清除

提示：如果终端中没有选中文本，Ctrl+C 会传递给终端执行中断命令的传统功能。

方法二：右键菜单
1. 在终端中用鼠标拖动选中一段文本
2. 在终端区域点击鼠标右键
3. 在弹出的菜单中选择"复制"
4. 选中的文本已复制到剪贴板，同时清除终端选区
```

**注意**:
- 没有选中文本时，复制选项为灰色不可用状态
- xterm.js 在右键点击时会清除选区，因此需要先拖动选中再右键

#### 3.5.2 粘贴到终端

```
方法一：快捷键
1. 确保目标面板处于活动状态
2. 按下 Ctrl+V
3. 剪贴板文本被写入终端 PTY

方法二：右键菜单
1. 在终端区域点击鼠标右键
2. 在弹出的菜单中选择"粘贴"
3. 剪贴板文本被写入终端 PTY，焦点自动恢复到终端
```

**注意**:
- 剪贴板为空时，粘贴选项为灰色不可用状态
- 粘贴后终端自动获得焦点，可继续输入

### 3.6 终端自动调整大小

Nexus 的终端会自动适应面板容器的尺寸，确保终端内容始终正确渲染。

**自动调整机制**:

| 触发条件 | 机制 | 说明 |
|---------|------|------|
| 窗口大小变化 | `window.resize` 事件 | 窗口缩放时终端同步调整 |
| 面板分割线拖动 | `ResizeObserver` | 面板比例调整时终端重新适应 |
| 面板首次创建 | `requestAnimationFrame` | 确保 DOM 渲染完成后再 fit |

**PTY 尺寸同步**:
- 终端调整大小的同时，PTY 的行列数也会同步更新
- 确保终端命令（如 `vim`、`top`）能够正确渲染布局

### 3.7 会话快照自动保存与恢复

Nexus 自动保存和恢复每个会话的终端布局状态，无需手动操作。

#### 3.7.1 自动保存时机

| 操作 | 保存内容 |
|------|---------|
| 创建第一个面板 | 初始布局树 |
| 新增面板（分屏） | 布局树 + 新面板节点 |
| 关闭面板 | 更新后的布局树 |
| 拖拽调整面板大小 | 各面板的 flex 比例 |

#### 3.7.2 快照恢复

**切换会话时**:
```
1. 优先检查内存缓存
   - 有缓存 → 瞬间恢复（PTy 进程保持活动）
   - 无缓存 → 从数据库读取快照
2. 重建所有面板的 PTY 进程
3. 恢复布局树和面板比例
4. 自动设置活动面板焦点
```

**应用启动时**:
```
1. 加载上次活动的会话
2. 从快照或缓存中恢复布局
3. 重建所有面板的终端实例
4. 应用启动后与上次关闭时状态一致
```

**内存缓存机制**:
- 每个会话的面板和布局缓存在内存中（`sessionsPanels` 和 `sessionsLayouts`）
- 切换会话时，当前会话数据先写入缓存
- 缓存命中时恢复速度极快，无需数据库查询
- 删除会话时自动清理对应缓存

## 附录二：设计决策记录

### ADR-001: 一对多面板模式

**日期**: 2026-03-27

**决策**: 采用一对多面板模式，即一个会话可以包含多个终端面板。

**对比方案**:
| 方案 | 描述 | 优缺点 |
|------|------|--------|
| 一对一（原方案） | 一个会话绑定一个面板 | 简单，但关闭面板后会话失效 |
| 一对多（新方案） | 一个会话可有多个面板 | 灵活，支持分屏，关闭面板不影响会话 |

**采用理由**:
1. 用户关闭所有面板后，可以重新添加面板，无需删除重建会话
2. 支持分屏功能，满足多任务并行需求
3. 符合现代终端复用器的设计理念

**实现要点**:
- 会话数据结构从 `panelId: string` 改为 `panelIds: string[]`
- 新增 `activePanelId` 字段记录当前聚焦的面板
- 布局模式决定新增面板时的排列方式

### ADR-002: 布局模式选择器

**日期**: 2026-03-27

**决策**: 顶部工具栏设置布局选择器，用户可选择"左右分屏"或"上下分屏"。

**功能说明**:
- **左右分屏**: 新增面板时横向排列，宽度平均分配
- **上下分屏**: 新增面板时纵向排列，高度平均分配

**实现方式**:
- 左右分屏：`flex-direction: row`
- 上下分屏：`flex-direction: column`
- 每个面板的宽度/高度 = `100% / 面板数量`

**用户操作流程**:
1. 点击"布局"按钮展开下拉菜单
2. 选择目标模式（左右/上下）
3. 点击"新增面板"按钮
4. 选择路径后创建新面板

**对比 tmux**:
| 操作 | tmux | Nexus |
|------|------|-------|
| 左右分屏 | `Ctrl+b %` | 布局选择"左右分屏" → 新增面板 |
| 上下分屏 | `Ctrl+b "` | 布局选择"上下分屏" → 新增面板 |

**优势**:
- 无需记忆快捷键
- 布局模式可切换，更符合 GUI 操作习惯

### ADR-003: 树形嵌套布局

**日期**: 2026-03-27

**决策**: 采用树形嵌套布局，支持嵌套分屏功能。

**核心理念**:
- 每次分割只针对**当前选中的面板**，而不是所有面板
- 使用树形数据结构表示嵌套布局
- 关闭面板后自动简化空容器
- 面板间的分割线可拖拽调整大小

**数据结构**:
```typescript
interface LayoutNode {
  type: 'horizontal' | 'vertical';
  children: LayoutChild[];
}
interface PanelNode {
  type: 'panel';
  id: string;
}
type LayoutChild = LayoutNode | PanelNode;
```

**实现要点**:
- 会话数据结构使用 `layout: LayoutNode` 字段表示布局树
- `LayoutRenderer` 组件递归渲染布局树
- 分割面板时更新布局树中对应节点
- 关闭面板时自动简化空容器节点
- `Resizer` 组件实现面板间拖拽调整大小

**视觉效果**:
```
场景：面板 1 | [面板 2 / 面板 3]

┌──────────────┬─────────────┐
│              │   面板 2     │
│   面板 1      ├─────────────┤
│              │   面板 3     │
└──────────────┴─────────────┘
```

**布局特性**:
- 支持任意深度的嵌套分割
- 面板间分割线可拖拽调整相对大小
- 调整后的比例自动保存到会话快照
