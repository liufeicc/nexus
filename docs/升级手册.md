# 版本升级操作手册（打包人员）

本文档说明：**当新版本涉及数据库结构变更时，打包人员在发布前需要做什么。**

---

## 什么时候需要操作？

当你的代码修改了以下内容之一时：

- 新增 / 删除 / 修改数据库表
- 新增 / 修改 / 删除索引、触发器
- 修改 `database.ts` 中 `initDatabase()` 内的 SQL

**如果只是修改了 UI、业务逻辑、配置项等，不涉及数据库变更，则无需操作。**

---

## 操作步骤

### 1. 递增数据库版本号

打开 `src/main/db/database.ts`，找到：

```typescript
const CURRENT_DB_VERSION = 1
```

将版本号 **加 1**（如改为 `2`）。

### 2. 在 `runMigrations` 中添加迁移逻辑

找到 `runMigrations` 函数中的注释区域：

```typescript
// 未来新增迁移时，在此追加：
// if (targetVersion >= 2) {
//   console.log('[Database] 执行迁移 2: xxx')
//   db.exec('CREATE TABLE ...')
// }
```

取消注释并编写实际的 SQL 变更，例如：

```typescript
if (targetVersion >= 2) {
  console.log('[Database] 执行迁移 2: 新增 user_preferences 表')
  db.exec(`
    CREATE TABLE IF NOT EXISTS user_preferences (
      id TEXT PRIMARY KEY,
      key TEXT NOT NULL,
      value TEXT NOT NULL
    );
  `)
}
```

> **注意**：
> - 使用 `IF NOT EXISTS` 保证幂等
> - 使用 `ALTER TABLE` 修改已有表结构
> - 迁移逻辑在 **事务** 中执行，确保原子性

### 3. 测试迁移逻辑

1. 用一个 **旧版本的数据库** 启动应用
2. 确认控制台输出类似：
   ```
   [Database] 数据库版本 1 -> 2，开始迁移...
   [Database] 执行迁移 2: 新增 user_preferences 表
   [Database] 迁移完成
   ```
3. 确认旧数据没有丢失，新功能正常工作

### 4. 构建发布包

按正常流程打包：

```bash
npm run build
```

electron-builder 会自动在输出目录生成 `latest.yml`，将其与安装包一起上传到你的更新服务器即可。
