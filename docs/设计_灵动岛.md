# 智能体 UI 设计方案 — 小松鼠

## 背景

用户需要一个智能体助手，以小松鼠形象出现在应用界面中。松鼠主要在标题栏上活动，通过传送门机制在不同面板间移动，主动观察用户操作并提醒，支持对话交互。

## 核心设计

### 角色形象：「小松鼠」🐿️

直接使用 emoji 🐿️ 作为角色（约 20x20px），位于标题栏内部（32px 高度的标题栏内）。

- **形象**：系统自带 🐿️ emoji，无需 SVG/图片资源
- **颜色**：跟随系统 emoji 渲染，不受主题色影响
- **优点**：零开发成本、跨平台兼容、表现力自然

### 表情与状态（通过 CSS 动画 + 叠加元素实现）

emoji 本身无法变形，通过以下组合表达不同状态：

| 状态 | emoji 表现 | 叠加元素 | CSS 效果 |
|------|-----------|---------|---------|
| idle | 🐿️ | 无 | 轻微上下浮动 |
| looking | 🐿️ | 无 | 左右轻微平移 |
| thinking | 🐿️ | 头顶 `...` | 静止 |
| speaking | 🐿️ | 无 | 对话气泡 |
| alert | 🐿️ | 头顶 `❗` | 轻微抖动 |
| error | 🐿️ | 无 | 摇头动画 |
| happy | 🐿️ | 无 | 跳跃弹跳 |
| sleeping | 🐿️ | 飘浮 `z` | 缓慢呼吸起伏 |

### 移动与传送机制

**待机位置**：左上角第一个面板的标题栏内

**移动到目标面板的流程**：

```
步骤1: 松鼠在左上角待机
┌─🐿️────────┬──────────────┐
│ 终端   ✕   │ 文件浏览器✕  │
├────────────┼──────────────┤
│            │              │

步骤2: 检测到文件浏览器有问题，松鼠向右跑
┌────────────┬──────────────┐
│ 终端   ✕   │🐿️文件浏览器✕│  ← 松鼠沿标题栏向右移动
├────────────┼──────────────┤
│            │              │

步骤3: 到达左边缘，出现漩涡传送门
┌────────────┬──────────────┐
│ 终端   ✕   │🌀文件浏览器✕ │  ← 标题栏左侧出现漩涡
├────────────┼──────────────┤
│            │              │

步骤4: 松鼠从漩涡中出现，开始说话+指点
┌────────────┬──────────────┐
│ 终端   ✕   │🐿️文件浏览器✕│  ← 松鼠在目标面板标题栏上
├────────────┼──────────────┤
│            │  这里有问题!  │  ← 对话气泡指向面板内容
│            │              │
```

**移动动画细节**：
- 沿标题栏水平移动：CSS `transform: translateX()` + `transition`
- 到达传送门位置时短暂停顿（200ms）
- 漩涡出现：CSS `@keyframes portalAppear`（缩放 + 旋转 + 透明度）
- 松鼠从漩涡中出现：从 `scale(0)` 放大到 `scale(1)` + 轻微弹跳

### 小动作（idle 状态）

- 轻微上下浮动（呼吸效果）
- 偶尔左右小范围平移（张望）
- 长时间不动后进入 sleeping 状态（缓慢呼吸 + `z` 飘浮）

### 对话气泡

- 从松鼠位置向下延伸，位于面板内容区域上方
- 圆角矩形 + CSS 三角指针（指向松鼠）
- 逐字打字机效果（30ms/字符）
- 最大宽度 280px，自动换行
- 5 秒后自动消失（可点击关闭）

### 点击交互

- 点击松鼠：打开聊天面板（右侧滑入，320x400px）
- 聊天面板：消息列表 + 输入框 + 发送按钮
- Phase 1 使用预设回复，Phase 2 对接 AI 后端

## 技术方案

| 项目 | 方案 |
|------|------|
| 角色渲染 | emoji 🐿️，零资源成本 |
| 水平移动 | CSS `transform: translateX()` + `transition` |
| 表情/小动作 | CSS `@keyframes` 动画（浮动、抖动、摇头等） |
| 传送门 | 纯 CSS（圆形 + 旋转渐变边框） |
| 覆盖层 | `position: fixed; inset: 0; pointer-events: none; z-index: 9000` |
| 状态管理 | 独立 Zustand store |
| 观察用户 | 订阅主 store + 监听 PTY 数据事件 |

## 实现阶段

### Phase 1: 基础 emoji 角色
创建 🐿️ emoji 角色组件、独立 store、覆盖层，角色出现在左上角标题栏，有 idle 浮动动画。

### Phase 2: 表情和状态动画
实现 8 种状态的 CSS 动画（浮动、抖动、摇头、弹跳等）、头顶叠加元素（`...`、`❗`、`z`）。

### Phase 3: 传送门和移动
实现沿标题栏水平移动、漩涡传送门出现/消失动画、松鼠从传送门中出现。

### Phase 4: 对话气泡和聊天面板
打字机效果、对话气泡向下延伸、聊天面板、预设 AI 回复。

### Phase 5: 自主行为引擎
订阅主 store 观察面板变化、监听终端输出检测错误、自动移动到目标面板标题栏、显示对话气泡。

### Phase 6: 打磨和 AI 对接（远期）
剩余动画、性能优化、Header 添加开关按钮、替换预设回复为真实 LLM 调用。

# 灵动岛（Dynamic Island）通知系统

## 背景

需要一个浮动通知组件，用于向用户展示系统状态、任务完成提示、错误警告等信息。要求：
- 显示在系统顶部，不被浏览器面板（WebContentsView）遮蔽
- 支持拖动，边界限制在窗口范围内
- 消息以打字机效果逐字显示
- 支持外部程序（如 Hermes Agent）通过进程间通信发送通知

## 架构演变

| 阶段 | 方案 | 结果 |
|------|------|------|
| V1 | 独立 BrowserWindow | 透明窗口不可见，拖动冲突，放弃 |
| V2 | 主窗口内 React 浮层 | `position: fixed` + `z-index: 100000`，成功 |

最终方案：纯 React 浮层组件，嵌入主窗口渲染进程，通过高 z-index 确保不被 `WebContentsView` 遮挡。

## 消息队列架构

### 整体流程

```
[外部程序 Python]                     [Electron 主进程]              [渲染进程]
       |                                    |                          |
       |  socket.connect() + send(JSON)     |                          |
       |  ~/.config/tview/notify.sock ────> │  UDS Server              |
       |                                    │  解析 JSON → 验证         |
       |                                    │  MessageQueueDAO.enqueue()│
       |                                    │  [status=pending]         |
       |                                    │                           |
       |                                    |  DynamicIsland 轮询       |
       |                                    |  ← dequeueOne() (0.5s)   |
       |                                    |  [status=reading → read]  |
       |                                    |                           |
       |                                    │  webContents.send() ──>   │
       |                                    │                           │
       |                                    │                Typewriter 显示
       |                                    │                5s 后收起
       |                                    │                恢复轮询下一条
```

### 消息状态流转

```
pending → reading → read  (正常流程)
pending → failed          (显示异常时)
```

1. **pending**：外部消息写入数据库，等待显示
2. **reading**：渲染进程取到消息，开始显示（原子操作，防止重复取）
3. **read**：打字机动画完成，标记已读
4. **failed**：显示异常时标记失败

## 技术方案

| 项目 | 方案 |
|------|------|
| 外部通信 | Unix Domain Socket（Linux/macOS）/ Named Pipe（Windows） |
| Socket 路径 | `~/.config/tview/notify.sock`（固定路径，不随开发/生产变化） |
| 消息持久化 | SQLite `message_queue` 表 |
| 取消息方式 | 渲染进程每 0.5 秒轮询数据库 |
| 打字机效果 | `TypewriterText` 组件，50ms/字符 |
| 停留时间 | 打字完成后停留 5 秒 |
| 拖动 | JS 鼠标事件 + 边界钳位（`Math.max/min`） |
| 样式 | 科幻 HUD 风格，Orbitron + Share Tech Mono 字体 |

## 关键实现细节

### 1. UDS 服务器（src/main/services/uds-server.ts）

- 使用 Node.js 内置 `net.createServer()`
- 接收 `\n` 分隔的 JSON 消息，支持批量发送
- 消息验证：title 必填、message 最长 5000 字、type 必须为合法值
- 启动失败不阻塞主进程，仅记录日志
- POSIX 下设置 socket 文件权限 `0o777`，允许其他用户连接
- 应用退出时自动清理 socket 文件

### 2. 消息队列 DAO（src/main/db/message-queue.dao.ts）

| 方法 | 说明 |
|------|------|
| `enqueue(title, message, type, duration?, icon?)` | 写入 pending 消息，返回自增 ID |
| `dequeueOne()` | 原子操作：取最老 pending 消息并标记为 reading |
| `markRead(id)` | 标记为已读（显示完成后调用） |
| `markFailed(id)` | 标记为失败（显示异常时调用） |
| `listByStatus(status)` | 查询指定状态的消息列表 |

使用 `db.transaction()` 保证 `dequeueOne` 的原子性，避免并发取到同一条消息。

### 3. 灵动岛组件（src/renderer/components/common/DynamicIsland.tsx）

**状态机**：

```
idle → showing → complete → hiding → idle
```

- **idle**：收起状态，只显示小指示器（36px 高度）
- **showing**：展开显示消息内容，打字机动画进行中
- **complete**：打字完成，停留 5 秒
- **hiding**：收起动画（0.4s 过渡）

**轮询机制**：
- 每 500ms 调用 `electronAPI.messageQueue.dequeue()`
- 正在显示消息时暂停轮询（`isPollingPausedRef`）
- 收起后恢复轮询

**高度自适应**：
- 使用 `max-height: 36px` → `max-height: 800px` 过渡
- `height: auto` 让内容自然撑开容器
- 打字时容器平滑增长，不会截断文字

**拖动边界钳位**：
```typescript
newX = Math.max(0, Math.min(newX, winW - ISLAND_WIDTH))
newY = Math.max(0, Math.min(newY, winH - currentHeight))
```

### 4. 打字机组件（src/renderer/components/common/TypewriterText.tsx）

- `setInterval` 逐字显示，50ms/字符
- 空文本时用 `setTimeout(0)` 延迟触发完成回调，避免渲染期间同步更新其他组件状态
- 闪烁光标 CSS 动画

### 5. 样式要点（src/renderer/styles/dynamic-island.css）

- `z-index: 100000` 确保高于所有 UI 元素
- `-webkit-app-region: no-drag` 覆盖标题栏拖动区域
- 透明背景 + 半深色内容区背景
- Orbitron + Share Tech Mono 科幻字体
- 4 种消息类型用不同颜色的左侧竖条区分（info 蓝、success 绿、warning 黄、error 红）

## Python 调用示例

```python
import socket, json
from pathlib import Path

def notify(title: str, message: str, msg_type: str = 'info'):
    sock = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
    sock.connect(Path.home() / '.config' / 'tview' / 'notify.sock')
    payload = {'title': title, 'message': message, 'type': msg_type}
    sock.sendall((json.dumps(payload, ensure_ascii=False) + '\n').encode('utf-8'))
    sock.close()

# 使用
notify("任务完成", "数据处理完毕", "success")
notify("警告", "磁盘空间不足", "warning")
notify("错误", "连接超时", "error")
```
