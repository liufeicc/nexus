## 1. 功能概述

### 1.1 什么是智能窗口？

智能窗口是 Nexus 内置的 AI 辅助功能，通过为每个终端面板配备独立的 AI Agent（称为**面板 Agent**），实时监终端输出、自动判断终端状态，并将分析结果以有趣的方式呈现给用户。

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| **终端输出监控** | 自动检测终端是否有新输出，积累并分析 |
| **等待输入检测** | 识别 shell 提示符，提示用户终端等待输入 |
| **错误检测** | 识别命令执行错误，及时通知用户 |
| **状态判断** | 通过 LLM 理解终端输出，判断当前执行状态 |
| **流式通知** | 以小松鼠动画 + 气泡的方式展示 AI 分析结果 |

### 1.3 架构简介

```
┌─────────────────────────────────────────────┐
│  面板 A                  面板 B              │
│  ┌──────┐               ┌──────┐            │
│  │  🐿️  │ 独立 Agent     │  🐿️  │ 独立 Agent │
│  │ 终端  │               │ 终端  │            │
│  └──────┘               └──────┘            │
│                                              │
│  主进程 (Main Process)                       │
│  ┌──────────────────────────────────────┐   │
│  │  LLM 调用（通义千问 / OpenAI / 自定义） │   │
│  │  API Key 加密存储                     │   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

**设计要点**：
- 每个面板的 Agent 独立运行，互不影响
- LLM 调用在主进程执行，API Key 不会暴露给渲染进程
- 面板 Agent 需要用户手动开启，避免不必要的 API 消耗

## 2. 配置 AI 服务

使用 AI 功能前，需要先配置 AI 服务商和 API Key。

### 2.1 打开 AI 配置面板

**操作步骤**：
```
1. 点击顶部标题栏右侧的 AI 图标（灯泡图标）
2. AI 配置面板从右侧滑入
```

**界面位置**：
```
┌─────────────────────────────────────────────┐
│  Nexus              [AI] [主题] [？] [-][□][×]│
│                         ↑                    │
│                      点击这里                 │
└─────────────────────────────────────────────┘
```

### 2.2 选择 AI 服务商

AI 配置面板打开后，首先看到**AI 服务商**选择框：

```
┌──────────────────────────────────────┐
│  AI 配置                        [✕]  │
├──────────────────────────────────────┤
│                                      │
│  AI 服务商                           │
│  ┌─────────────────────────────┐    │
│  │ 通义千问 (DashScope)      ▼ │    │
│  └─────────────────────────────┘    │
│                                      │
│  可选服务商：                         │
│  - 通义千问 (DashScope)              │
│  - OpenAI                            │
│  - 自定义                            │
│                                      │
└──────────────────────────────────────┘
```

| 服务商 | 说明 |
|--------|------|
| **通义千问 (DashScope)** | 阿里云的大语言模型服务，推荐国内用户使用 |
| **OpenAI** | OpenAI 的 GPT 系列模型，需要海外 API 访问权限 |
| **自定义** | 输入自定义 API 端点地址，兼容 OpenAI 格式的 API |

**操作**：点击下拉框，选择目标服务商。

### 2.3 填写 API Key

```
  API Key
  ┌─────────────────────────────────────┐
  │ ••••••••••••••••••••••••••••••    │  ← 密码输入框
  └─────────────────────────────────────┘
```

**操作**：
```
1. 在 API Key 输入框中粘贴你的 API Key
2. 输入框以密码模式显示（掩码字符）
3. API Key 将通过 Electron safeStorage 加密存储
```

**获取 API Key**：

| 服务商 | 获取地址 |
|--------|----------|
| 通义千问 | https://dashscope.console.aliyun.com/ |
| OpenAI | https://platform.openai.com/api-keys |

### 2.4 选择模型

```
  模型
  ┌─────────────────────────────────────┐
  │ qwen-plus                       ▼ │
  └─────────────────────────────────────┘
```

根据选择的服务商，模型选项会自动更新：

**通义千问可用模型**：

| 模型 | 特点 | 适用场景 |
|------|------|----------|
| qwen-turbo | 速度快，成本低 | 日常监控、快速判断 |
| qwen-plus | 平衡速度与质量 | 推荐使用 |
| qwen-max | 质量最高，速度较慢 | 复杂错误分析 |

**OpenAI 可用模型**：

| 模型 | 特点 |
|------|------|
| gpt-4o | 最强模型，质量好 |
| gpt-4o-mini | 快速、低成本 |

### 2.5 自定义 API 端点

当服务商选择**自定义**时，会额外显示 API 端点输入框：

```
  API 端点
  ┌─────────────────────────────────────┐
  │ https://api.example.com/v1         │
  └─────────────────────────────────────┘
```

**说明**：
- 输入兼容 OpenAI 格式的 API 端点 URL
- 适用于本地部署的 LLM（如 Ollama、vLLM 等）
- 端点需要支持 `/chat/completions` 接口

### 2.6 保存配置

```
┌──────────────────────────────────────┐
│  ...                                │
├──────────────────────────────────────┤
│  未配置                      [保存]  │
│  ↑状态提示                 ↑保存按钮  │
└──────────────────────────────────────┘
```

**操作**：
```
1. 填写完所有必填项后，点击"保存"按钮
2. 状态提示从"未配置"变为"已配置"（绿色显示）
3. AI 配置面板自动关闭
4. 配置信息加密保存到本地
```

**状态提示**：

| 状态 | 显示 | 含义 |
|------|------|------|
| 未配置 | 灰色文字"未配置" | 尚未配置 AI 服务 |
| 已配置 | 绿色文字"已配置" | 配置成功，可以使用 AI 功能 |
| 调用中 | 蓝色文字"调用中" | 正在调用 LLM API |
| 错误 | 红色文字"错误" | API 调用失败，请检查配置 |

## 3. 开启面板 AI 监控

### 3.1 面板小松鼠图标

每个面板的标题栏右侧、关闭按钮左侧，有一个小松鼠图标（🐿️）：

```
┌─────────────────────────────────────────────┐
│ [●] bash - ~/projects    🐿️  [关闭]        │  ← 面板标题栏
│                              ↑               │
│                           小松鼠图标          │
├─────────────────────────────────────────────┤
│                                             │
│              终端内容区域                     │
│                                             │
└─────────────────────────────────────────────┘
```

**未激活状态**：
- 小松鼠图标呈灰度显示（半透明）
- 鼠标悬停时放大并高亮

### 3.2 开启面板 Agent

**场景一：未配置 AI 时点击小松鼠**

```
操作：点击小松鼠图标
结果：弹出提示 "请先配置 AI 服务商"
```

**场景二：已配置 AI 时点击小松鼠**

```
操作步骤：
1. 确保已配置 AI 服务商（见第 2 章）
2. 点击面板标题栏的小松鼠图标
3. 小松鼠图标变为彩色，播放弹跳动画
4. 面板 Agent 启动，开始监控终端输出
```

**激活效果**：
```
激活前：🐿️（灰度、半透明）
激活后：🐿️（彩色、正常透明度，播放弹跳动画）
```

**关闭面板 Agent**：
```
操作：再次点击已激活的小松鼠图标
效果：小松鼠恢复灰度状态，停止监控
```

### 3.3 小松鼠状态含义

面板 Agent 运行期间，小松鼠会根据终端状态展示不同的行为和表情：

| 状态 | 表情 | 含义 |
|------|------|------|
| **监控中** | 🐿️ | 小松鼠安静地坐着，偶尔摆动尾巴，表示正在监控终端输出 |
| **分析中** | 🐿️💭 | 小松鼠头顶出现思考气泡，抱着坚果沉思，表示正在分析终端输出 |
| **汇报中** | 🐿️📡 | 小松鼠拿着对讲机，表示正在向总控汇报（如果有总控 Agent） |
| **执行中** | 🐿️⚡ | 小松鼠奔跑或跳跃，表示正在执行命令 |
| **等待输入** | 🐿️👀 | 小松鼠探头张望，竖起耳朵，表示终端等待用户操作 |
| **检测到错误** | 🐿️⚠️ | 小松鼠举着警告牌，尾巴竖起，表情紧张 |

## 4. 小松鼠输出气泡

### 4.1 气泡显示方式

面板 Agent 激活后，小松鼠旁边会出现一个气泡，用于显示 AI 分析结果：

```
┌─────────────────────────────────────────────┐
│ [●] bash - ~/projects    🐿️  [关闭]        │
│                           ↑                  │
│                        ┌──┴──┐              │
│                        │ ▲   │  ← 小箭头     │
│                        ├─────┤              │
│                        │ AI分析结果          │  ← 气泡内容
│                        └─────┘              │
├─────────────────────────────────────────────┤
│              终端内容区域                     │
└─────────────────────────────────────────────┘
```

**气泡特性**：
- 位置：小松鼠右侧偏下，带箭头指向小松鼠
- 宽度：约 280px
- 最大高度：200px，超出后可滚动
- 背景：与主题适配的深色背景
- 关闭：取消面板 Agent 时气泡自动消失

### 4.2 流式输出效果

气泡中的信息以**流式逐行**的方式显示，每条新消息带入场动画：

```
时间线：

T+0s  气泡出现：
      ┌──────────────────────────┐
      │ 检测到终端输出...        │  ← 第 1 行，淡入
      └──────────────────────────┘

T+1s  新增第 2 行：
      ┌──────────────────────────┐
      │ 检测到终端输出...        │
      ├──────────────────────────┤
      │ 正在分析终端状态...      │  ← 第 2 行，从左侧滑入
      └──────────────────────────┘

T+3s  新增第 3 行：
      ┌──────────────────────────┐
      │ 检测到终端输出...        │
      ├──────────────────────────┤
      │ 正在分析终端状态...      │
      ├──────────────────────────┤
      │ 命令执行完毕，返回提示符  │  ← 第 3 行，从左侧滑入
      └──────────────────────────┘
```

**动画效果**：
- 新行出现时从左侧轻微滑入（`translateX(-4px) → translateX(0)`）
- 透明度从 0 渐变到 1
- 动画持续 0.3 秒

### 4.3 气泡中的信息类型

气泡支持三种信息类型的显示，各有不同视觉风格：

**普通信息**（白色/主题色文字）：
```
│ 检测到终端输出，正在分析...  │
│ 命令执行完毕，返回 shell 提示符 │
│ git pull 已完成              │
```

**流式信息**（灰色文字，表示进行中）：
```
│ 正在分析终端输出...          │
│ 等待命令执行完成...           │
│ 编译进度 65%...              │
```

**错误信息**（红色文字，加粗）：
```
│ ⚠️ 检测到错误: docker build 失败 │
│ ⚠️ 错误: command not found: npm  │
│ ⚠️ 编译失败: 3 errors, 5 warnings│
```

## 5. 设置智能助手开关

### 5.1 打开设置窗口

**操作步骤**：
```
1. 点击顶部标题栏右侧的齿轮图标（设置图标）
2. 设置窗口以模态对话框形式弹出
```

**界面位置**：
```
┌─────────────────────────────────────────────────────┐
│  Nexus              [主题] [⚙] [？] [-][□][×]     │
│                         ↑                            │
│                      点击这里                         │
└─────────────────────────────────────────────────────┘
```

**关闭方式**：
| 方式 | 操作 |
|------|------|
| 按钮 | 点击窗口底部的"关闭"按钮 |
| 快捷键 | 按 `Esc` 键 |
| 外部点击 | 点击窗口外部区域 |

### 5.2 智能助手开关操作

设置窗口打开后，显示智能助手开关：

```
┌──────────────────────────────────────┐
│  ⚙ 设置                              │
├──────────────────────────────────────┤
│                                      │
│  智能体                              │
│  智能助手启动或关闭        [ 开关 ]   │
│                                      │
├──────────────────────────────────────┤
│                          [关闭]       │
└──────────────────────────────────────┘
```

**开启智能助手**：
```
1. 打开设置窗口
2. 点击"智能助手"右侧的开关，将其拨到开启位置（右侧）
3. 开关变为高亮色（主题色），表示已启用
4. 配置自动保存，可直接关闭设置窗口
```

**关闭智能助手**：
```
1. 打开设置窗口
2. 点击"智能助手"右侧的开关，将其拨到关闭位置（左侧）
3. 开关变为灰色，表示已禁用
4. 配置自动保存，可直接关闭设置窗口
```

### 5.3 开关状态说明

| 状态 | 视觉表现 | 含义 |
|------|----------|------|
| **开启** | 滑块在右侧，背景为高亮色 | 智能助手已启用，将定时扫描操作日志 |
| **关闭** | 滑块在左侧，背景为灰色 | 智能助手已禁用，不会执行任何操作 |

> **注意**：所有设置修改后会自动保存到本地数据库，下次启动 Nexus 时自动恢复上次状态。

## 7. 常见问题

### 7.1 AI 配置相关

#### Q1: 支持哪些 AI 服务商？

目前支持三种：
- **通义千问 (DashScope)**：阿里云服务，推荐国内用户
- **OpenAI**：需要海外网络访问
- **自定义**：兼容 OpenAI 格式的任何 API 端点

#### Q2: API Key 安全吗？

安全。API Key 使用 Electron safeStorage 系统级加密存储在本地，不会发送到除 AI 服务商 API 之外的任何地方。

#### Q3: 可以切换 AI 服务商吗？

可以。随时点击顶部标题栏的 AI 图标重新打开配置面板，修改服务商、API Key 或模型。

### 7.2 面板 Agent 相关

#### Q4: 每个面板都需要单独开启 AI 吗？

是的。每个面板的 Agent 是独立的，需要用户手动为每个需要监控的面板开启。这样可以避免不必要的 API 调用消耗。

#### Q5: 开启 AI 监控会消耗多少 Token？

面板 Agent 只发送终端输出的纯文本（去除 ANSI 转义码），每次最多 3000 字符或 1000 行。典型的短命令（如 `ls`、`cd`）只消耗几十个 Token。长时间运行的命令（如 `docker build`）可能分段发送，总消耗取决于输出量。

#### Q6: 点击小松鼠提示"请先配置 AI 服务商"怎么办？

说明你还没有配置 AI 服务。点击顶部标题栏右侧的 AI 图标，按照第 2 章的步骤配置服务商和 API Key。

#### Q7: 小松鼠一直是灰色的怎么办？

灰色表示面板 Agent 未激活。点击小松鼠图标即可开启。如果点击后没有反应，请先确认 AI 配置是否正确。

### 7.3 气泡相关

#### Q8: 气泡不显示怎么办？

可能的原因：
1. 面板 Agent 未正确激活 → 重新点击小松鼠
2. 终端没有新输出 → 执行一条命令触发检测
3. AI 调用失败 → 检查状态栏 AI 指示灯是否为红色

#### Q9: 气泡内容太多看不完怎么办？

气泡最大高度 200px，超出后可滚动。旧的消息会保留在气泡顶部，向上滚动即可查看。
