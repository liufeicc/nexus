## 1. 功能概述

**文件面板**是 Nexus 的第二种面板类型（区别于终端面板），专注于**文件浏览**和**轻量文件操作**。它与终端面板共存于同一布局系统中，让你在终端旁边直接浏览和管理文件，无需切换到系统文件管理器。

面板采用**大图标横向排列**的文件浏览器视图，配合面包屑导航和右键菜单，实现高效的目录浏览和文件操作。

### 核心价值

| 价值点 | 描述 |
|--------|------|
| 减少上下文切换 | 不需要在终端和系统文件管理器之间来回切换 |
| 直观文件浏览 | 大图标视图一目了然，快速识别文件和文件夹 |
| 轻量文件操作 | 复制、粘贴、删除等常用操作无需敲命令 |

### 使用场景

| 场景 | 说明 |
|------|------|
| 查看项目结构 | 快速浏览项目目录树，了解文件组织 |
| 定位文件路径 | 通过图标视图快速找到目标文件并复制路径 |
| 文件整理 | 复制、粘贴、删除文件和文件夹 |
| 终端配合工作 | 左侧文件面板浏览文件，右侧终端执行命令 |

### 2.2 工具栏

窗口顶部工具栏包含面板创建入口：

```
┌─────────────────────────────────────────────────────────┐
│  [+ 文件] [+ 终端]              [主题] [？] [-] [□] [×]│
│   ↑新增文件面板  ↑新增终端面板                           │
└─────────────────────────────────────────────────────────┘
```

| 元素 | 功能 |
|------|------|
| [+ 文件] | 新增文件面板，点击后直接创建，默认根目录为当前工作目录 |
| [+ 终端] | 新增终端面板，点击后创建新的终端会话 |

### 2.3 面板标题栏

文件面板顶部显示简洁的标题栏：

```
┌─────────────────────────────────────┐
│  ~/project                    [×] │  ← 面板标题栏
├─────────────────────────────────────┤
│  ... 文件内容区域 ...                │
└─────────────────────────────────────┘
```

| 元素 | 功能 |
|------|------|
| 根目录路径 | 动态显示创建时指定的根目录路径 |
| [×] 关闭按钮 | 关闭该文件面板（无需确认，因为不依赖 PTY 进程） |

### 2.4 面包屑导航栏

位于标题栏下方，显示从根目录到当前目录的完整路径：

```
┌─────────────────────────────────────────────────────────────┐
│  📁 root  >  📁 src  >  📁 components  >  📁 file-browser   │
│   [点击]     [点击]     [点击]        [当前，不可点击]       │
└─────────────────────────────────────────────────────────────┘
```

| 元素 | 功能 |
|------|------|
| 根目录段 | 点击跳转回根目录 |
| 中间路径段 | 点击跳转到对应层级目录 |
| 当前目录段 | 显示为当前所在位置，不可点击 |
| `>` 分隔符 | 视觉分隔，表示层级关系 |

**路径编辑模式**: 点击面包屑导航栏的空白区域可进入路径编辑模式，支持手动输入路径、Tab 键自动补全、上下键导航建议项。详见 [3.4 节](#34-面包屑导航与路径编辑)。

### 2.5 文件状态栏

位于文件面板最底部，显示已打开的文件标签，支持快速切换：

```
┌──────────────────────────────────────────────────────────────────┐
│  [📁 目录]  [📄 index.ts]  [📄 README.md ×]  [📄 config.json ×]  │
│   ↑回到网格    ↑当前查看      ↑其他文件      ↑过长时截断... ×      │
└──────────────────────────────────────────────────────────────────┘
```

| 元素 | 功能 |
|------|------|
| 目录标签 | 始终显示在状态栏最左侧，点击退出文件查看，回到文件网格浏览（不关闭已打开的文件标签） |
| 文件标签 | 显示文件名，点击可切换查看该文件 |
| 当前高亮 | 正在查看的文件以强调色背景高亮 |
| 关闭按钮 | 每个文件标签上显示 ×，点击关闭该文件 |
| 空状态 | 未打开文件时显示"未打开文件"提示 |

**行为说明**:
- 双击文件网格中的文件 → 进入查看状态，同时添加到状态栏
- 点击状态栏其他文件 → 切换查看内容
- 点击当前文件的关闭按钮 → 回到文件网格（浏览状态）
- 点击"目录"标签 → 回到文件网格，但不关闭已打开的文件标签
- 状态栏文件过多时支持横向滚动

### 2.6 文件列表区域（大图标视图）

面板主体区域，以大图标卡片形式展示当前目录下的所有条目：

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐         │
│  │          │  │          │  │          │  │          │         │
│  │    📂    │  │    📂    │  │    📂    │  │    📄    │         │
│  │          │  │          │  │          │  │          │         │
│  │  compo   │  │  hooks   │  │  store   │  │  index   │         │
│  │  nents   │  │          │  │          │  │  .ts     │         │
│  │          │  │          │  │          │  │          │         │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘         │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 图标尺寸 | 约 90x90px |
| 排列方式 | 横向排列，自动换行 |
| 排序规则 | 文件夹在前、文件在后，各自按名称字母排序 |
| 选中态 | 卡片高亮边框/背景 |
| 双击文件夹 | 进入该目录，面包屑导航同步更新 |
| 双击文件 | 打开文件查看器（支持文本/图片/PDF） |
| 隐藏文件 | 默认不显示（以 `.` 开头的文件） |

**文件图标颜色**:
- 📂 文件夹：蓝色
- 📄 文件：灰色
- 🔗 符号链接：橙色

## 3. 功能操作详解

### 3.1 创建文件面板

文件面板和终端面板各自拥有独立的创建按钮。

**操作步骤**:

```
方法一：通过工具栏按钮
1. 点击顶部工具栏的 [+ 文件] 按钮
2. 系统立即创建文件面板，默认根目录为当前工作目录
3. 创建后以大图标显示根目录下的文件列表

方法二：右键菜单分屏
1. 在任意面板空白处右键点击
2. 选择"水平分屏"或"垂直分屏"
3. 在二级菜单中选择"文件面板"
4. 系统创建文件面板并添加到对应布局位置
```

### 3.2 浏览目录

**双击进入目录**:

```
1. 在文件列表中找到目标文件夹图标
2. 快速双击该图标
3. 面板内容刷新，显示该文件夹的内容
4. 面包屑导航自动更新
```

**返回上级目录**:
- 点击面包屑中的上一级路径段
- 或使用快捷键 `Backspace`

**目录自动刷新**: 当外部程序对当前目录进行了增删操作时，文件面板会自动刷新显示最新内容。

### 3.3 文件查看

文件面板支持**查看状态**和**浏览状态**两种模式。双击文件即可进入查看状态。

#### 3.3.1 打开文件查看

```
1. 在文件网格中双击目标文件图标
2. 文件网格区域替换为文本查看视图
3. 文件状态栏添加该文件的图标标签并高亮
4. 面包屑导航保持不变
```

**支持查看的文件类型**:

| 类别 | 扩展名 | 渲染方式 |
|------|--------|---------|
| 编程语言 | ts, tsx, js, jsx, py, go, rs, java, c, cpp, h, sh, bash 等 | `<pre>` 文本 |
| 配置/数据 | json, yaml, yml, toml, xml, ini, csv, sql 等 | `<pre>` 文本 |
| 标记/文档 | md, txt, html, css, scss, less, svg 等 | `<pre>` 文本 |
| 特殊文件 | .gitignore, Dockerfile, Makefile, .env, .prettierrc 等 | `<pre>` 文本 |
| 图片 | png, jpg, jpeg, gif, webp, bmp, ico, svg, tiff, tif | `<img>` 自适应缩放 |
| PDF | pdf | pdf.js 渲染到 Canvas，支持翻页 |

不支持的文件类型双击时会提示"不支持查看此类型的文件"。

#### 3.3.2 切换查看文件

```
方法一：通过状态栏切换
1. 当前处于文件查看状态
2. 点击文件状态栏中的其他文件图标
3. 文本查看区内容刷新为该文件内容
4. 高亮标签跟随移动

方法二：回到文件网格重新选择
1. 点击状态栏的"目录"标签
2. 回到文件网格（浏览状态），已打开的文件标签保留
3. 双击其他文件重新打开
```

#### 3.3.3 关闭文件查看

```
方法一：关闭当前文件
1. 点击文件状态栏中当前文件的关闭按钮（×）
2. 文本查看区隐藏，回到文件网格
3. 如果有其他已打开文件，自动切换到第一个；否则完全回到网格

方法二：退出查看模式
1. 按 Escape 键
2. 回到文件网格，同时关闭当前查看的文件
```

#### 3.3.4 图片文件查看

双击图片文件（PNG、JPG、GIF、WebP、BMP、ICO、SVG、TIFF 等格式）时，文件查看器会自动渲染图片：

```
┌─────────────────────────────────────────┐
│  🖼️  screenshot.png              [×]  │  ← 文件图标变为图片图标（绿色）
├─────────────────────────────────────────┤
│                                         │
│         ┌───────────────────┐           │
│         │                   │           │
│         │    [图片预览]     │           │
│         │                   │           │
│         └───────────────────┘           │
│          图片自适应容器大小               │
│                                         │
└─────────────────────────────────────────┘
```

| 特性 | 说明 |
|------|------|
| 自适应缩放 | 图片自动适配容器大小，保持宽高比 |
| 居中显示 | 图片在查看器中水平垂直居中 |
| 禁止拖拽 | 图片不可拖拽保存，避免误操作 |

#### 3.3.5 PDF 文件查看

双击 PDF 文件时，文件查看器使用 pdf.js 引擎在浏览器端渲染 PDF 内容：

```
┌──────────────────────────────────────────────┐
│  📄  document.pdf    ‹ 1 / 5 ›        [×]  │  ← 图标变为红色，显示翻页控制
├──────────────────────────────────────────────┤
│                                              │
│        ┌────────────────────────────┐        │
│        │                            │        │
│        │     [PDF 页面内容]         │        │
│        │                            │        │
│        │                            │        │
│        └────────────────────────────┘        │
│         白底阴影，模拟纸张效果                 │
│                                              │
└──────────────────────────────────────────────┘
```

| 特性 | 说明 |
|------|------|
| 页码显示 | 显示当前页码和总页数（如 "1 / 5"） |
| 上一页 | 点击 `‹` 按钮或快捷键切换到上一页 |
| 下一页 | 点击 `›` 按钮或快捷键切换到下一页 |
| 自适应缩放 | PDF 页面根据容器宽度自动缩放，确保完整显示 |
| 首尾页限制 | 第一页时上一页按钮禁用，最后一页时下一页按钮禁用 |

**注意事项**:
- PDF 渲染依赖 pdf.js 库，首次打开 PDF 文件时需要从 CDN 加载 worker 脚本
- 网络不可用时，PDF 可能无法正常渲染
- 大型 PDF 文件（100+ 页）首次加载可能稍有延迟

### 3.4 面包屑导航与路径编辑

#### 3.4.1 点击跳转

```
1. 查看面包屑导航栏，显示从根目录到当前目录的路径
2. 点击任意一级路径段（除当前目录外）
3. 面板内容跳转到该层级目录
4. 文件列表自动刷新
```

#### 3.4.2 路径编辑模式

```
1. 点击面包屑导航栏的空白区域（非路径段位置）
2. 进入路径编辑模式，出现文本输入框
3. 输入路径，系统会实时显示子目录建议列表
4. 按 Tab 键自动补全当前输入
5. 按 ↑↓ 键在建议列表中上下选择
6. 按 Enter 键确认选择并跳转到该目录
7. 按 Escape 键退出编辑模式
```

### 3.5 文件选择（单击/框选/全选）

#### 3.5.1 单击选中

- 单击文件/文件夹卡片 → 选中该项（高亮显示）
- 单击空白区域 → 取消所有选中

#### 3.5.2 Ctrl/Cmd + 点击多选

- 按住 `Ctrl`（Linux/Windows）或 `Cmd`（Mac）点击多个卡片 → 逐项添加/移除选中
- 已选中的卡片再次 Ctrl/Cmd + 点击 → 取消该项选中

#### 3.5.3 框选（鼠标拖拽）

```
1. 在文件网格空白区域按下鼠标左键
2. 拖动鼠标绘制蓝色半透明矩形选区
3. 松开水滴，选区内的所有文件/文件夹被选中
4. 选区中心点落在卡片上的条目即被选中
```

#### 3.5.4 全选与取消

| 操作 | 效果 |
|------|------|
| `Ctrl+A` / `Cmd+A` | 全选当前目录下所有条目 |
| `Escape` | 取消所有选中 |
| 点击空白区域 | 取消所有选中 |

### 3.7 文件复制、剪切与粘贴

#### 3.7.1 复制文件

```
方法一：快捷键
1. 选中一个或多个文件/文件夹
2. 按 Ctrl+C（Linux/Windows）或 Cmd+C（Mac）
3. 提示"已复制 N 个对象"

方法二：右键菜单
1. 右键选中文件
2. 点击"复制"
3. 提示"已复制 N 个对象"
```

#### 3.7.2 剪切文件

```
方法一：快捷键
1. 选中一个或多个文件/文件夹
2. 按 Ctrl+X（Linux/Windows）或 Cmd+X（Mac）
3. 提示"已剪切 N 个对象"
4. 被剪切的文件在文件列表中显示为半透明状态

方法二：右键菜单
1. 右键选中文件
2. 点击"剪切"
3. 提示"已剪切 N 个对象"
```

**剪切行为说明**:

| 场景 | 行为 |
|------|------|
| 同级目录粘贴 | 直接移动到目标位置，不生成副本 |
| 不同级目录粘贴 | 先复制到目标位置，成功后将源文件移入系统回收站 |
| 目标文件已存在 | 跳过该文件，不覆盖 |

**半透明提示**: 被剪切但尚未粘贴的文件/文件夹会以 50% 透明度显示，方便识别当前处于剪切状态的对象。粘贴或复制操作后该状态自动清除。

#### 3.7.3 粘贴文件

```
方法一：快捷键
1. 导航到目标目录
2. 按 Ctrl+V（Linux/Windows）或 Cmd+V（Mac）
3. 执行粘贴

方法二：右键菜单
1. 在目标目录空白处右键
2. 点击"粘贴"
3. 执行粘贴
```

**粘贴行为说明**:

| 场景 | 行为 |
|------|------|
| 复制模式 - 同级目录粘贴 | 自动生成副本，名称递增为 `文件名(副本 1)`、`文件名(副本 2)` 等 |
| 复制模式 - 不同级目录粘贴 | 目标已存在时弹出覆盖确认对话框，可选择是否覆盖 |
| 剪切模式 - 同级目录粘贴 | 直接移动，不生成副本 |
| 剪切模式 - 不同级目录粘贴 | 移动到目标位置，源文件被移入系统回收站 |
| 多个文件粘贴 | 逐个执行，完成后统一提示结果 |

#### 3.7.4 内部剪贴板

文件复制和剪切使用**内部剪贴板**（Zustand store 中维护），与系统剪贴板独立。内部剪贴板仅在当前应用内有效，关闭应用后清空。

内部剪贴板有两种模式：

| 模式 | 触发方式 | 粘贴行为 |
|------|---------|---------|
| **复制模式** | Ctrl+C / 右键"复制" | 粘贴时生成副本，源文件保留 |
| **剪切模式** | Ctrl+X / 右键"剪切" | 粘贴时移动源文件，源文件被删除 |

执行新的复制或剪切操作会替换内部剪贴板的内容和模式。

### 3.8 文件删除（移入回收站）

#### 3.8.1 删除文件

```
方法一：快捷键
1. 选中一个或多个文件/文件夹
2. 按 Delete 键
3. 弹出确认对话框"确定要将 N 个对象移入回收站吗？"
4. 点击确认后删除

方法二：右键菜单
1. 右键选中文件
2. 点击"删除"
3. 弹出确认对话框，点击确认后删除
```

#### 3.8.2 删除后恢复

删除的文件进入**系统回收站**，可通过操作系统的回收站/废纸篓功能恢复：
- **macOS**: Finder 侧边栏的"废纸篓"
- **Windows**: 桌面"回收站"
- **Linux**: `~/.local/share/Trash` 目录

删除后文件面板会自动刷新当前目录列表。

## 5. 常见问题解答

### 5.1 入门问题

#### Q1: 文件面板和终端面板有什么区别？

| 对比项 | 终端面板 | 文件面板 |
|--------|---------|---------|
| 主要功能 | 运行命令、执行脚本 | 浏览文件、轻量文件操作 |
| 依赖 | 依赖 PTY 进程 | 独立，不依赖 PTY |
| 关闭时 | 终止 PTY 进程（需确认） | 直接关闭，无需确认 |
| 交互方式 | 键盘输入命令 | 鼠标点击、右键菜单导航 |

#### Q2: 什么时候用文件面板，什么时候用终端命令？

| 场景 | 推荐方式 | 原因 |
|------|---------|------|
| 查看文件结构 | 文件面板 | 大图标视图更直观 |
| 快速复制路径 | 文件面板 | 右键即可，无需输入命令 |
| 大量文件批量操作 | 终端命令 | `find`、`rsync` 等更高效 |
| 深层目录浏览 | 文件面板 | 面包屑导航比反复 `cd` 更快捷 |

### 5.2 文件操作问题

#### Q3: 如何复制文件路径？

在文件/文件夹图标上右键 → 选择"复制路径" → 路径已复制到系统剪贴板 → 可在任意位置粘贴使用。
如果右键在空白区域，则复制当前目录路径。

#### Q4: 粘贴按钮为什么是灰色的？

粘贴按钮灰色表示不可用，原因可能是：
- 内部剪贴板为空（尚未复制或剪切任何文件）

解决方法：先执行"复制文件"或"剪切文件"操作。

#### Q5: 复制和剪切有什么区别？

| 对比项 | 复制（Ctrl+C） | 剪切（Ctrl+X） |
|--------|---------------|---------------|
| 源文件 | 保留在原位置 | 粘贴后被移入系统回收站 |
| 同级目录粘贴 | 生成副本 | 直接移动 |
| 不同级目录粘贴 | 复制一份到目标位置 | 移动到目标位置 |
| 视觉提示 | 无 | 被剪切文件显示为半透明 |

根据需求选择：需要保留源文件用复制，需要移动文件用剪切。

#### Q5: 同级目录下粘贴会发生什么？

这取决于剪贴板模式：
- **复制模式**（Ctrl+C）：自动生成副本，文件名递增为 `文件名(副本 1)`、`文件名(副本 2)` 等
- **剪切模式**（Ctrl+X）：直接移动到目标位置，不生成副本

#### Q6: 不同级目录下粘贴，目标文件已存在怎么办？

- **复制模式**：弹出覆盖确认对话框，可以选择是否覆盖
- **剪切模式**：跳过该文件，不覆盖

#### Q11: 可以新建文件夹或文本文件吗？

可以。在文件面板任意位置右键 → 选择"新建" → 选择"新建文件夹"或"新建文本"即可。
如果当前目录已存在同名项目，系统会自动添加序号后缀如 `新建文件夹 (1)`、`新建文本 (1).txt`。

#### Q12: 为什么右键在文件夹上点击"新建"，新文件没有创建在该文件夹内？

右键在文件夹图标上新建时，新文件创建在该文件夹的**同级目录**下。如需在子文件夹内创建，请先双击进入该文件夹后再右键新建。

### 5.3 文件删除问题

#### Q13: 删除的文件能恢复吗？

可以。删除的文件移入**系统回收站**，不是永久删除。可通过以下方式恢复：
- **macOS**: 打开 Finder → 点击侧边栏"废纸篓" → 右键文件选择"放回原处"
- **Windows**: 打开桌面"回收站" → 右键文件选择"还原"
- **Linux**: 从 `~/.local/share/Trash/files` 目录移回文件

### 5.4 文件预览问题

#### Q14: 可以预览图片和 PDF 文件吗？

可以。双击以下类型文件即可在文件查看器中预览：

| 类型 | 格式 | 渲染方式 |
|------|------|---------|
| 图片 | PNG、JPG、GIF、WebP、BMP、ICO、SVG、TIFF | 图片自适应居中显示，保持宽高比 |
| PDF | PDF | 使用 pdf.js 渲染到 Canvas，支持翻页 |

文件图标颜色也会根据类型变化：文本=主题色、图片=绿色、PDF=红色。

#### Q15: PDF 文件打开后无法显示内容？

PDF 渲染依赖 pdf.js 的 worker 脚本，需要从 CDN 加载。如果网络不可用或 CDN 无法访问，PDF 将无法正常渲染。请检查网络连接。

### 5.5 性能问题

#### Q16: 目录下文件很多时会不会卡？

- 文件数 < 200：正常渲染，无明显延迟
- 文件数 200-500：列表支持滚动，可能有轻微延迟
- 文件数 > 500：建议改用终端命令（如 `ls`）浏览

### 5.6 权限与异常

#### Q17: 根目录被外部删除了怎么办？

面板会检测到路径不存在，显示提示并提供"重新选择目录"按钮。点击后可选择新的根目录。

#### Q18: 遇到"无法访问"提示怎么办？

"无法访问"表示当前用户对该目录无读权限。解决方法：
- 检查目录权限设置
- 尝试使用 `sudo` 提升权限（在终端中）
- 选择其他有权限的目录

## 设计决策记录

### ADR-001: 大图标视图 vs 传统列表视图

**日期**: 2026-04-10

**决策**: 采用大图标横向排列视图，而非传统的列表式逐行显示。

**采用理由**:
1. 大图标视图更直观，文件/文件夹一目了然
2. 横向排列 + 自动换行，适配不同面板宽度
3. 与现代操作系统（macOS Finder、Windows 资源管理器）的图标视图一致

### ADR-002: 无工具栏按钮

**日期**: 2026-04-10

**决策**: 不在面板中显示复制、粘贴、新建等工具栏按钮，所有操作通过右键菜单和快捷键完成。

**采用理由**:
1. 减少视觉干扰，让大图标视图更干净
2. 右键菜单和快捷键是用户熟悉的文件操作方式
3. 面板空间有限，优先展示文件内容

### ADR-003: 文件面板不依赖 PTY

**日期**: 2026-04-09

**决策**: 文件面板拥有独立的生命周期，不依赖 PTY 进程。

**采用理由**:
1. 文件浏览不需要 PTY，减少资源占用
2. 关闭文件面板时不需要确认，操作更轻量
3. 快照恢复时无需重建 PTY，速度更快

### ADR-004: 文件查看器采用全屏替换模式

**日期**: 2026-04-11

**决策**: 双击文件时，文件网格区域替换为文本查看视图（全屏模式），而非分屏或弹窗模式。

**采用理由**:
1. 原型阶段简化实现，避免嵌套布局复杂性
2. 文件查看时网格不需要同时显示，减少视觉干扰
3. 通过底部状态栏可快速切换已打开文件
4. 后续可扩展为分屏模式（用户反馈驱动）

### ADR-005: 单文件查看模式

**日期**: 2026-04-11

**决策**: 文件面板同时只查看一个文件内容，通过状态栏切换。

**采用理由**:
1. 简化状态管理，避免多标签页复杂性
2. 文件面板定位为文件浏览器，非代码编辑器
3. 状态栏已提供快速切换能力

### ADR-006: 图片/PDF 预览采用浏览器端渲染

**日期**: 2026-04-12

**决策**: 图片通过 IPC 读取为 base64 后用 `<img>` 标签显示；PDF 使用 pdf.js 在浏览器端渲染到 Canvas。

**采用理由**:
1. 图片 base64 方案简单直接，利用现有 IPC 基础设施
2. pdf.js 是成熟的浏览器端 PDF 渲染库，无需服务端支持
3. 避免依赖系统原生图片/PDF 查看器，保证跨平台一致性
4. worker 通过 CDN 加载，不增加应用包体积

**权衡**:
- PDF 渲染依赖网络（CDN worker），离线环境可能无法使用
- 大型 PDF 文件性能不如原生查看器
