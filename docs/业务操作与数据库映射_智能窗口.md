## 1. AI 配置操作

### 1.1 AI 配置面板操作

| 界面操作 | 触发时机 | 数据库操作 | 说明 |
|---------|---------|-----------|------|
| 点击 AI 配置图标 | 打开面板 | `ConfigDAO.get('aiConfig')` | 读取已保存的 AI 配置，恢复表单状态 |
| 点击 AI 配置图标 | 打开面板 | `ConfigDAO.get('aiModelOptions')` | 读取模型选项数据，填充服务商下拉 |
| 切换 AI 服务商 | 选择服务商 | 无 | 内存操作：根据服务商动态更新模型下拉选项 |
| 点击保存 | 保存配置 | `ConfigDAO.save('aiConfig', config)` | 保存 AI 配置到数据库 |
| ESC / 点击遮罩 | 关闭面板 | 无 | 纯 UI 操作 |

**AI 配置保存流程**:
```
1. 用户点击顶部标题栏的 AI 配置图标（齿轮图标）

2. 面板从右侧滑入，加载已保存的配置
   ConfigDAO.get('aiConfig')
   → SELECT value FROM configs WHERE key = 'aiConfig'
   → 如果存在，填充表单字段（provider、apiKey、model、customEndpoint）
   → 状态显示"已配置"（绿色）
   → 如果不存在，状态显示"未配置"（灰色）

3. 同时读取模型选项数据
   ConfigDAO.get('aiModelOptions')
   → 读取各服务商对应的可用模型列表
   → 填充到服务商切换逻辑中

4. 用户选择服务商、填写 API Key、选择模型

5. 点击"保存"按钮
   ConfigDAO.save('aiConfig', {
     provider: 'dashscope',
     apiKey: 'sk-xxxxx',
     model: 'qwen-plus',
     customEndpoint: ''
   })
   → INSERT OR REPLACE INTO configs (key, value, updated_at)
     VALUES ('aiConfig', '{"provider":"dashscope",...}', strftime('%s', 'now'))

6. UI 反馈
   - 状态从"未配置"变为绿色"已配置"
   - 600ms 后自动关闭面板
```

**AI 配置项说明**:

| 配置字段 | 控件类型 | 必填 | 说明 |
|---------|---------|-----|------|
| provider | 下拉选择 | 是 | 服务商：dashscope / openai / custom |
| apiKey | 密码输入框 | 是 | 掩码显示，MVP 阶段明文存储到数据库 |
| model | 下拉/文本输入 | 是 | 预设服务商用下拉，自定义用文本输入 |
| customEndpoint | 文本输入 | 自定义时必填 | 仅在 provider='custom' 时显示 |

### 1.2 AI 配置状态指示灯

| 状态 | 颜色 | 触发条件 |
|------|------|---------|
| 灰色（未配置） | var(--text-muted) | `ConfigDAO.get('aiConfig')` 返回 null |
| 绿色（已配置） | #4ade80 | 存在有效的 aiConfig 记录 |
| 蓝色（调用中） | #60a5fa | LLM 请求进行中（后续迭代） |
| 红色（错误） | #f87171 | LLM 请求失败（后续迭代） |

### 1.3 智能窗口关联触发场景

| 界面操作 | 触发条件 | 行为 |
|---------|---------|------|
| 点击面板小松鼠（未配置 AI） | aiConfig 不存在 | 小松鼠说话提示"请先配置 AI 服务商信息" → 自动打开 AI 配置面板 |

## 2. 配置管理汇总表 — 智能窗口

| 配置项 | 保存操作 | 读取操作 | 删除操作 | 值类型 | 说明 |
|-------|---------|---------|---------|-------|------|
| aiConfig | `ConfigDAO.save('aiConfig', config)` | `ConfigDAO.get('aiConfig')` | `ConfigDAO.delete('aiConfig')` | JSON 对象 | AI 服务配置（provider、apiKey、model、customEndpoint） |
| aiModelOptions | 初始化时写入 | `ConfigDAO.get('aiModelOptions')` | `ConfigDAO.delete('aiModelOptions')` | JSON 对象 | 各服务商的可用模型选项列表 |

### aiConfig 值结构

```json
{
  "provider": "dashscope",
  "apiKey": "sk-xxxxx",
  "model": "qwen-plus",
  "customEndpoint": ""
}
```

### aiModelOptions 值结构

```json
{
  "dashscope": [
    { "value": "qwen-turbo", "label": "qwen-turbo（快速）" },
    { "value": "qwen-plus", "label": "qwen-plus（平衡，推荐）" },
    { "value": "qwen-max", "label": "qwen-max（高质量）" }
  ],
  "openai": [
    { "value": "gpt-4o", "label": "gpt-4o" },
    { "value": "gpt-4o-mini", "label": "gpt-4o-mini" }
  ],
  "custom": [
    { "value": "", "label": "自定义模型" }
  ]
}
```

## 3. 数据初始化流程

### 3.1 模型选项初始化

主进程数据库初始化时，检查 `aiModelOptions` 是否存在，不存在则写入默认值：

```
1. 应用启动，数据库初始化完成

2. 检查默认模型选项
   ConfigDAO.get('aiModelOptions')
   → SELECT value FROM configs WHERE key = 'aiModelOptions'

3. 如果返回 null（首次启动）
   ConfigDAO.save('aiModelOptions', DEFAULT_MODEL_OPTIONS)
   → INSERT OR REPLACE INTO configs (key, value, updated_at)
     VALUES ('aiModelOptions', '{"dashscope":[...],...}', strftime('%s', 'now'))

4. 如果已存在，跳过（保留用户自定义的修改）
```

**初始化代码位置**: `src/main/index.ts` 或数据库初始化模块中，在 `ConfigDAO` 初始化后执行。

## 4. 完整业务流程示例

### 4.1 首次启动 + 配置 AI

```
1. 应用启动

2. 数据库初始化
   ConfigDAO.get('aiModelOptions')
   → 返回 null（首次启动）

   ConfigDAO.save('aiModelOptions', {
     dashscope: [{ value: 'qwen-turbo', label: '...' }, ...],
     openai: [{ value: 'gpt-4o', label: '...' }, ...],
     custom: [{ value: '', label: '自定义模型' }]
   })
   → 写入默认模型选项

3. 用户点击 AI 配置图标
   ConfigDAO.get('aiConfig')
   → 返回 null（未配置）
   → 面板显示"未配置"状态

4. 用户填写配置并保存
   ConfigDAO.save('aiConfig', {
     provider: 'dashscope',
     apiKey: 'sk-xxxxx',
     model: 'qwen-plus',
     customEndpoint: ''
   })
   → INSERT OR REPLACE INTO configs (key, value, updated_at)
     VALUES ('aiConfig', '{"provider":"dashscope",...}', strftime('%s', 'now'))

5. 状态变为"已配置"，面板自动关闭
```

### 4.2 非首次启动（恢复配置）

```
1. 应用启动

2. 数据库初始化
   ConfigDAO.get('aiModelOptions')
   → 已存在，跳过写入

3. 用户点击 AI 配置图标
   ConfigDAO.get('aiConfig')
   → 返回 { provider: 'dashscope', apiKey: 'sk-xxxxx', model: 'qwen-plus', customEndpoint: '' }

   → 表单自动填充：
     - 服务商下拉选中"通义千问 (DashScope)"
     - API Key 显示掩码
     - 模型下拉选中"qwen-plus"
   → 状态显示绿色"已配置"
```

### 4.3 切换服务商时更新模型选项

```
1. 用户在 AI 配置面板中将服务商从"通义千问"切换为"OpenAI"

2. 内存操作（不涉及数据库）
   → 从已加载的 aiModelOptions 中读取 openai 对应的模型列表
   → 更新模型下拉选项为 gpt-4o / gpt-4o-mini
   → 隐藏自定义 API 端点字段

3. 用户点击保存
   ConfigDAO.save('aiConfig', {
     provider: 'openai',
     apiKey: 'sk-xxxxx',
     model: 'gpt-4o',
     customEndpoint: ''
   })
   → 更新数据库中的 aiConfig 记录
```

### 4.4 使用自定义服务商

```
1. 用户选择"自定义"服务商

2. 内存操作
   → 模型下拉切换为文本输入框
   → 显示自定义 API 端点字段

3. 用户填写：
   - API 端点: https://api.example.com/v1
   - 模型: my-custom-model
   - API Key: sk-xxxxx

4. 点击保存
   ConfigDAO.save('aiConfig', {
     provider: 'custom',
     apiKey: 'sk-xxxxx',
     model: 'my-custom-model',
     customEndpoint: 'https://api.example.com/v1'
   })
```
