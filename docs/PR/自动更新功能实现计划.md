# Nexus 自动更新功能实现计划

## 背景

Nexus 当前没有自动更新机制，用户需要手动下载新版本。需要实现：
- 默认 GitHub Releases 发布，失败时自动回退到国内 HTTPS CDN
- 启动时静默检查 + 手动检查更新按钮（两者结合）
- 全量更新（不实现差量）
- 支持 Windows (NSIS)、Linux (AppImage)、macOS (DMG)

## 实现步骤

### 1. 安装依赖
```bash
npm install electron-updater
```

### 2. electron-builder.json 添加 publish
```json
{
  "publish": [
    {
      "provider": "github",
      "owner": "liufei",
      "repo": "Nexus",
      "releaseType": "release"
    },
    {
      "provider": "generic",
      "url": "https://nexus-cdn.example.com/releases/",
      "channel": "latest"
    }
  ]
}
```
两个 provider 写在 electron-builder 中用于构建时生成 `latest-*.yml` 文件。实际的双源切换在 `update-service.ts` 代码中实现。

### 3. 新增 IPC 频道
`src/core/constants/ipc-channels.ts` 中添加：
- `AUTO_UPDATER_CHECK` — 渲染进程 → 主进程：触发检查
- `AUTO_UPDATER_DOWNLOAD` — 渲染进程 → 主进程：开始下载
- `AUTO_UPDATER_INSTALL` — 渲染进程 → 主进程：安装并重启
- `AUTO_UPDATER_STATE` — 主进程 → 渲染进程：状态变更事件
- `AUTO_UPDATER_PROGRESS` — 主进程 → 渲染进程：下载进度
- `AUTO_UPDATER_ERROR` — 主进程 → 渲染进程：错误通知

将 `AUTO_UPDATER_STATE`、`AUTO_UPDATER_PROGRESS`、`AUTO_UPDATER_ERROR` 加入 `NON_HANDLE_CHANNELS`。

### 4. 创建核心服务 `src/main/services/update-service.ts`
- **单例模式**，与现有 `DatabaseService`、`PtyService` 一致
- **状态机**：`idle → checking → available → downloading → downloaded → install`
- **双源回退逻辑**：
  1. 先调用 `autoUpdater.setFeedURL()` 配置 GitHub
  2. 调用 `autoUpdater.checkForUpdates()`
  3. 如果失败，捕获错误，再用 `setFeedURL()` 切换为 CDN 源重试
- **事件绑定**：监听 `update-available`、`download-progress`、`update-downloaded`、`error`，通过 `webContents.send()` 通知渲染进程
- **启动时自动检查**：`initialize()` 中 `setTimeout(() => this.checkForUpdates({ silent: true }), 5000)`，延迟 5 秒避免与启动竞争
- **开发模式跳过**：`process.env.NODE_ENV === 'development'` 时完全禁用

### 5. 创建 IPC 处理器 `src/main/ipc/handlers/update.ts`
注册 4 个 handler：
- `AUTO_UPDATER_CHECK` → 调用 `checkForUpdates()`
- `AUTO_UPDATER_DOWNLOAD` → 调用 `downloadUpdate()`
- `AUTO_UPDATER_INSTALL` → 调用 `installAndRestart()`
- `AUTO_UPDATER_SETTING` → 返回启用状态和当前 state

在 `src/main/ipc/ipc-handlers.ts` 中引入并调用 `registerUpdateHandlers()`。

### 6. 主进程初始化
`src/main/index.ts` 的 `initializeWindowServices()` 中：
```typescript
import { UpdateService } from '../services/update-service'
const updateService = UpdateService.getInstance()
updateService.setMainWindow(window)
updateService.initialize()
```

### 7. 预加载 API
`src/main/preload.ts` 中暴露 `autoUpdater` 对象：
- `checkForUpdates()`, `downloadUpdate()`, `installAndRestart()`, `getState()`
- `onStateChange(callback)`, `onProgress(callback)`, `onError(callback)` — 均返回清理函数

### 8. 类型定义
`src/renderer/electron-api.d.ts` 中添加 `UpdateState` 接口和 `autoUpdater` API 定义。

### 9. AboutModal UI 更新
更新 `src/renderer/components/common/AboutModal.tsx`，在版本信息下方添加：

| 状态 | 显示 |
|------|------|
| `idle` | "检查更新" 按钮 |
| `checking` | 加载动画 + "正在检查..." |
| `available` | "发现新版本 vX.X.X" + 下载按钮 |
| `downloading` | 进度条 + 百分比 |
| `downloaded` | "更新已就绪" + 安装重启按钮 |
| `not-available` | "已是最新版本" ✓ |
| `error` | 错误信息（红色）+ 重试按钮 |

- 组件挂载时调用 `getState()` 初始化状态
- 通过 `useEffect` 注册 `onStateChange`、`onProgress`、`onError` 监听
- Linux .deb 用户显示"此包类型不支持自动更新"

### 10. i18n 文案
在所有 locale JSON 文件中添加 `update.*` 键：
`checkForUpdates`, `checking`, `updateAvailable`, `downloading`, `downloaded`, `upToDate`, `installAndRestart`, `download`, `retry`, `error`, `notSupported`, `releaseNotes`, `sourceGithub`, `sourceCdn`

## 验证

1. **开发模式验证**：启动 `npm run dev`，AboutModal 显示"开发模式下更新已禁用"
2. **生产构建验证**：`npm run build` 生成安装包 + `latest-*.yml`
3. **模拟更新**：手动修改版本号后构建，将产物放到本地测试服务器
4. **双源回退测试**：断网或用无效 GitHub URL 测试 CDN 回退
5. **平台覆盖**：至少在 Windows 上验证 NSIS 完整流程
