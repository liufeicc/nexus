# 应用嵌入面板（App-Embed）功能实现计划

> **文档版本**: v1.0
> **创建日期**: 2026-06-10
> **面板类型**: `app-embed` —— 在 Nexus 面板中嵌入任意原生桌面程序窗口
> **目标平台**: Linux (X11) / Windows / macOS
> **项目版本**: Nexus 1.0.0 · Electron 30 + React 18 + TypeScript 5

---

## 一、需求概述

### 1.1 功能目标

在 Nexus 终端应用中新增第四种面板类型 **app-embed**，用于在面板中嵌入任意原生桌面程序（如 gedit、VS Code、文件管理器等）的窗口，实现跨平台的"应用嵌入"功能。

### 1.2 使用场景

- 在 Nexus 工作区中直接嵌入代码编辑器（如 gedit、sublime）
- 嵌入文件管理器（如 nautilus、thunar）查看项目文件
- 嵌入图形工具（如 gimp、inkscape）处理设计资源
- 嵌入任意 GUI 程序，统一管理多个工作窗口

### 1.3 交互方式

- **嵌入式**：程序窗口显示在面板内部，用户可直接操作
- 面板标题栏显示程序名称，支持关闭、拖拽、分割等操作
- 支持快照保存/恢复，重启后自动恢复嵌入的程序

## 二、技术方案

### 2.1 架构分层

```
┌─────────────────────────────────────────────────────────────────────┐
│  Renderer (React)                                                    │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │ AppEmbedPanel.tsx   透明覆盖层 + placeholder div              │   │
│  │   ↕ useAppEmbedLifecycle / useAppEmbedViewSync hooks         │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                         ↕ IPC (contextBridge)                       │
├─────────────────────────────────────────────────────────────────────┤
│  Main (Electron)                                                    │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │ AppEmbedService (singleton)                                   │   │
│  │   ├── 进程管理：spawn / kill / health-check                  │   │
│  │   ├── 窗口句柄发现：XID / HWND / NSWindow                   │   │
│  │   └── 嵌入控制：reparent / SetParent / CGSPrivate           │   │
│  └──────────────────────────────────────────────────────────────┘   │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │ FFI 层 (koffi)                                                │   │
│  │   ├── Linux:   Xlib (XReparentWindow / XEmbed)               │   │
│  │   ├── Windows: user32.dll (SetParent / SetWindowLong)        │   │
│  │   └── macOS:   ApplicationServices (屏幕捕获 + 事件注入)     │   │
│  └──────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 设计原则

| 原则 | 说明 |
|------|------|
| **最小侵入** | 新增面板类型，仅追加式修改现有文件（类型联合、快照字段、LayoutRenderer 分支），不改既有逻辑 |
| **平台分层** | FFI 层暴露统一 API（`embed` / `resize` / `focus` / `detach`），内部按平台分发 |
| **进程隔离** | 所有原生调用在主进程执行；渲染进程只通过 IPC 发指令，不直接触碰原生模块 |
| **优雅降级** | macOS 不支持窗口嵌入时自动降级为"屏幕捕获镜像 + 键盘/鼠标注入"，或弹出提示 |
| **可观察性** | 嵌入式窗口的生命周期事件（启动/就绪/崩溃/退出）通过 IPC 事件推送给渲染进程 |

### 2.3 FFI 方案选型

采用 **koffi** 作为 FFI 库，原因：
- 纯 JavaScript 实现，无需编译原生模块
- 支持 Electron 版本升级时无需重新编译
- 性能损失约 10%，对窗口嵌入场景可接受
- 跨平台 API 调用简洁

**依赖安装**：
```bash
npm install koffi --save
```

### 2.4 平台技术方案

#### Linux (X11) — 完全支持 ✅

**核心机制：XEmbed / Reparent**

```
1. 子进程 spawn：child_process.spawn(cmd, args, { detached: true })
2. 窗口发现：
   - 监听 XCreateWindow 事件（XEventLoop）
   - 或用 xdotool search --pid <PID> 取 XID（简单但有 200~500ms 延迟）
   - 推荐：xdotool 作为 fallback，主方案用 Xlib 监听 CreateNotify
3. 嵌入：
   - 获取 Electron BrowserWindow 的 XID（nativeWindowHandle()）
   - 在目标区域创建一个 XEmbed Socket 窗口（child of Electron XID）
   - XReparentWindow(targetXID, socketXID, 0, 0)
   - 或使用 XEmbed 协议：XSendEvent + _XEMBED 消息
4. 跟随调整：
   - 监听 Electron 容器 resize → XResizeMove socket 窗口
5. 焦点与输入：
   - XSetInputFocus 转发
   - 键盘/鼠标事件天然由 X 服务器路由到子窗口
6. 关闭：
   - XDestroyWindow 或向子进程发 SIGTERM
```

**FFI 调用示例**：
```typescript
import koffi from 'koffi'

const X11 = koffi.load('libX11.so.6')
const XOpenDisplay = X11.func('Display* XOpenDisplay(char* display)')
const XReparentWindow = X11.func('int XReparentWindow(Display* display, Window w, Window parent, int x, int y)')
// ...
```

**依赖**：`libx11-6`, `libxext6`（运行时系统自带）
**Wayland 注意**：Wayland 下 XWayland 兼容层可以工作，但纯 Wayland 窗口无法嵌入。检测 `XDG_SESSION_TYPE`，非 X11 时给提示。

#### Windows — 完全支持 ✅

**核心机制：SetParent Win32 API**

```
1. 子进程 spawn：child_process.spawn(cmd, args, { windowsHide: false })
2. 窗口发现：
   - EnumWindows + GetWindowThreadProcessId 匹配 PID
   - 等待 IsWindowVisible() 为 TRUE（轮询，超时 5s）
3. 嵌入：
   - 获取宿主 HWND：browserWindow.getNativeWindowHandle()
   - SetWindowLongPtr(childHWND, GWL_STYLE, WS_CHILD | WS_VISIBLE)
   - SetParent(childHWND, hostHWND)
   - SetWindowPos(childHWND, HWND_TOP, 0, 0, w, h, SWP_FRAMECHANGED)
4. 跟随调整：
   - 监听 hostHWND 的 WM_SIZE 消息，MoveWindow(childHWND, ...)
5. 焦点：
   - SetFocus(childHWND) 或 SetActiveWindow
6. 关闭：
   - PostMessage(childHWND, WM_CLOSE, 0, 0)
   - 超时后 TerminateProcess
```

**FFI 调用示例**：
```typescript
const user32 = koffi.load('user32.dll')
const SetParent = user32.func('HWND SetParent(HWND hWndChild, HWND hWndNewParent)')
const SetWindowLongPtr = user32.func('LONG_PTR SetWindowLongPtrA(HWND hWnd, int nIndex, LONG_PTR dwNewLong)')
// ...
```

**依赖**：`user32.dll` 系统自带
**高 DPI**：嵌入前调用 `SetProcessDPIAware()`，防止窗口尺寸错位。

#### macOS — 降级支持 ⚠️

macOS 不原生支持窗口嵌入。提供两种方案：

**方案 B：屏幕捕获 + 事件注入（推荐，稳定）**
```
1. 子进程 spawn 正常启动
2. 发现窗口：
   - AXUIElementCreateApplication(pid)
   - 遍历 AXWindows 获取窗口位置/尺寸
3. 捕获：
   - CGWindowListCreateImage 截取窗口区域（每帧 ~5ms）
   - 或 screen.getSources({ types: ['window'] }) + thumbnail
   - 将帧绘制到 Electron 的 <canvas> 或 <img> 中
4. 输入注入：
   - AXUIElementSetAttributeValue 设置焦点
   - CGEventPost 注入键盘/鼠标事件
5. 帧率：30fps 足以满足大部分 GUI 应用
```

**macOS 实现策略**：
- v1.0 先采用方案 B（屏幕捕获）
- 在 UI 上明确标注 "macOS 为镜像模式，存在轻微延迟"
- 提供配置项关闭某些应用嵌入（避免权限问题）

```typescript
    ipcRenderer.invoke(IPC_CHANNELS.APP_EMBED_SET_BOUNDS, panelId, bounds),

  // 事件监听

    const listener = (_: any, data: any) => cb(data)
    ipcRenderer.on(IPC_CHANNELS.APP_EMBED_STATUS_CHANGED, listener)
    return () => ipcRenderer.removeListener(IPC_CHANNELS.APP_EMBED_STATUS_CHANGED, listener)

    const listener = (_: any, data: any) => cb(data)
    ipcRenderer.on(IPC_CHANNELS.APP_EMBED_FRAME_UPDATE, listener)
    return () => ipcRenderer.removeListener(IPC_CHANNELS.APP_EMBED_FRAME_UPDATE, listener)
  },
  onProcessExited: (cb: (data: { panelId: string; exitCode: number }) => void) => {
    const listener = (_: any, data: any) => cb(data)
    ipcRenderer.on(IPC_CHANNELS.APP_EMBED_PROCESS_EXITED, listener)
```

## 五、主进程服务设计

### 5.1 新增 `src/main/services/app-embed.service.ts`

```typescript
import { spawn, ChildProcess } from 'child_process'
import { BrowserWindow } from 'electron'
import { IPC_CHANNELS } from '../../core/constants/ipc-channels'
import type { AppEmbedStatusEvent, CreateAppEmbedParams } from '../../core/types/app-embed'

interface EmbedInstance {
  panelId: string
  process: ChildProcess
  pid: number
  /** 平台特定嵌入控制器 */
  controller: PlatformEmbedController
  mirrored: boolean
  /** macOS 镜像模式的帧定时器 */
  frameTimer?: NodeJS.Timeout
}

interface PlatformEmbedController {
  /** 等待子窗口出现 */
  waitForWindow(timeoutMs?: number): Promise<void>
  /** 将子窗口嵌入到宿主 */
  embed(hostHWND: Buffer, bounds: { x: number; y: number; width: number; height: number }): void
  /** 调整嵌入窗口尺寸 */
  resize(bounds: { x: number; y: number; width: number; height: number }): void
  /** 转移焦点 */
  focus(): void
  /** 释放嵌入（不杀进程） */
  detach(): void
  /** 杀进程 + 释放 */
  destroy(): void
}

export class AppEmbedService {
  private static instance: AppEmbedService | null = null
  private embeds = new Map<string, EmbedInstance>()
  private mainWindow: BrowserWindow | null = null
  private ffiModule: any = null  // FFI 模块（延迟加载）

  static getInstance(): AppEmbedService { /* ... */ }

  setMainWindow(window: BrowserWindow): void { /* ... */ }

  /** 延迟加载 FFI 模块 */
  private getFFI(): any {
    if (!this.ffiModule) {
      this.ffiModule = require('./ffi/embed-ffi')
    }
    return this.ffiModule
  }

  async create(params: CreateAppEmbedParams): Promise<{ pid: number; mirrored: boolean }> {
    const { panelId, executablePath, args = [], cwd } = params
    this.broadcast(panelId, { status: 'spawning' })

    const child = spawn(executablePath, args, {
      cwd,
      detached: false,
      windowsHide: false,
      stdio: 'ignore',
    })

    const mirrored = process.platform === 'darwin'
    const controller = this.createController(process.platform, child.pid!)

    const instance: EmbedInstance = { panelId, process: child, pid: child.pid!, controller, mirrored }
    this.embeds.set(panelId, instance)

    try {
      await controller.waitForWindow(5000)
      this.broadcast(panelId, { status: 'waiting-window' })

      const hostHWND = this.mainWindow!.getNativeWindowHandle()
      controller.embed(hostHWND, { x: 0, y: 0, width: 800, height: 600 })

      if (mirrored) {
        this.startFrameCapture(instance)
      }

      this.broadcast(panelId, { status: 'embedded', pid: child.pid })
    } catch (err: any) {
      this.broadcast(panelId, { status: 'error', errorMessage: err.message })
    }

    child.on('exit', (code) => {
      this.broadcast(panelId, { status: 'exited' })
      this.sendEvent(IPC_CHANNELS.APP_EMBED_PROCESS_EXITED, { panelId, exitCode: code ?? -1 })
      this.cleanupInstance(panelId)
    })

    return { pid: child.pid!, mirrored }
  }

  async resize(panelId: string, width: number, height: number): Promise<void> {
    const inst = this.embeds.get(panelId)
    if (!inst) return
    inst.controller.resize({ x: 0, y: 0, width, height })
  }

  async setBounds(panelId: string, bounds: { x: number; y: number; width: number; height: number }): Promise<void> {
    const inst = this.embeds.get(panelId)
    if (!inst) return
    inst.controller.resize(bounds)
  }

  async focus(panelId: string): Promise<void> {
    this.embeds.get(panelId)?.controller.focus()
  }

  async destroy(panelId: string): Promise<void> {
    const inst = this.embeds.get(panelId)
    if (!inst) return
    inst.controller.destroy()
    if (inst.frameTimer) clearInterval(inst.frameTimer)
    this.embeds.delete(panelId)
  }

  /** 应用退出时批量清理 */
  destroyAll(): void {
    for (const panelId of this.embeds.keys()) {
      this.destroy(panelId)
    }
  }

  private createController(platform: string, pid: number): PlatformEmbedController {
    const ffi = this.getFFI()
    if (platform === 'linux') return new ffi.LinuxEmbedController(pid)
    if (platform === 'win32') return new ffi.WinEmbedController(pid)
    if (platform === 'darwin') return new ffi.MacEmbedController(pid)
    throw new Error(`Unsupported platform: ${platform}`)
  }

  /** macOS 帧捕获 */
  private startFrameCapture(inst: EmbedInstance): void {
    inst.frameTimer = setInterval(async () => {
      try {
        const frame: string = await this.getFFI().captureWindowFrame(inst.pid)
        this.sendEvent(IPC_CHANNELS.APP_EMBED_FRAME_UPDATE, { panelId: inst.panelId, dataUrl: frame })
      } catch { /* 窗口消失时静默 */ }
    }, 33)  // ~30fps
  }

  private broadcast(panelId: string, partial: Partial<AppEmbedStatusEvent>): void {
    this.sendEvent(IPC_CHANNELS.APP_EMBED_STATUS_CHANGED, { panelId, ...partial })
  }

  private sendEvent(channel: string, data: any): void {
    this.mainWindow?.webContents.send(channel, data)
  }

  private cleanupInstance(panelId: string): void {
    const inst = this.embeds.get(panelId)
    if (!inst) return
    if (inst.frameTimer) clearInterval(inst.frameTimer)
    this.embeds.delete(panelId)
  }
}
```

### 6.2 `AppEmbedPanel.tsx` 骨架

```tsx
import React, { useRef, useEffect, useState } from 'react'
import type { AppEmbedPanel as AppEmbedPanelState } from '../../store'
import { useAppEmbedLifecycle } from './useAppEmbedLifecycle'
import { useAppEmbedViewSync } from './useAppEmbedViewSync'
import { AppEmbedPlaceholder } from './AppEmbedPlaceholder'

interface Props {
  panelId: string
  initialState: AppEmbedPanelState
}

export function AppEmbedPanel({ panelId, initialState }: Props) {
  const containerRef = useRef<HTMLDivElement>(null)
  const { status, errorMessage, mirrored } = useAppEmbedLifecycle(panelId, initialState)
  const { frameDataUrl } = useAppEmbedViewSync(panelId, containerRef, mirrored)

  const isReady = status === 'embedded'

  return (
    <div
      ref={containerRef}
      className="app-embed-panel"
      tabIndex={0}
      onFocus={() => window.electronAPI.appEmbed.focus(panelId)}
      style={{
        position: 'relative',
        width: '100%',
        height: '100%',
        overflow: 'hidden',
        background: '#000',
      }}
    >
      {!isReady && (
        <AppEmbedPlaceholder status={status} errorMessage={errorMessage} />
      )}
      {/* macOS 镜像模式：显示捕获帧 */}
      {mirrored && frameDataUrl && (
        <img
          src={frameDataUrl}
          alt="embedded app"
          style={{ width: '100%', height: '100%', objectFit: 'contain' }}
        />
      )}
      {/* Linux/Windows：嵌入窗口直接覆盖在 container 上方，由原生代码负责定位 */}
      {isReady && !mirrored && (
        <div style={{ position: 'absolute', inset: 0 }} />
      )}
    </div>
  )
}
```

### 6.3 `useAppEmbedViewSync.ts`（尺寸同步）

```typescript
import { useEffect, useRef, useState } from 'react'

export function useAppEmbedViewSync(
  panelId: string,
  containerRef: React.RefObject<HTMLDivElement>,
  mirrored: boolean | undefined,
) {
  const [frameDataUrl, setFrameDataUrl] = useState<string | null>(null)
  const observerRef = useRef<ResizeObserver | null>(null)

  useEffect(() => {
    const container = containerRef.current
    if (!container) return

    // 监听容器尺寸变化，同步到嵌入窗口
    const syncBounds = () => {
      const rect = container.getBoundingClientRect()
      window.electronAPI.appEmbed.setBounds(panelId, {
        x: Math.round(rect.left),
        y: Math.round(rect.top),
        width: Math.round(rect.width),
        height: Math.round(rect.height),
      })
    }

    observerRef.current = new ResizeObserver(syncBounds)
    observerRef.current.observe(container)
    syncBounds()

    // macOS 帧更新
    let unsubFrame: (() => void) | undefined
    if (mirrored) {
      unsubFrame = window.electronAPI.appEmbed.onFrameUpdate(({ panelId: pid, dataUrl }) => {
        if (pid === panelId) setFrameDataUrl(dataUrl)
      })
    }

    return () => {
      observerRef.current?.disconnect()
      unsubFrame?.()
    }
  }, [panelId, mirrored])

  return { frameDataUrl }
}
```

## 十一、风险评估与缓解

| 风险 | 等级 | 影响 | 缓解措施 |
|------|------|------|----------|
| **FFI 性能损耗** | 低 | 窗口操作延迟略高 | 对性能敏感场景可切换至原生 addon |
| **macOS 镜像模式延迟** | 中 | 用户体验不如原生嵌入 | 动态调整帧率；明确标注限制 |
| **Wayland 不支持 XEmbed** | 中 | Linux Wayland 用户无法使用 | 检测后提示切换到 X11 会话 |
| **外部程序崩溃** | 中 | 面板变成空白 | 监听 exit 事件，显示"已退出"占位 UI，支持一键重启 |
| **macOS 权限要求** | 中 | 需要辅助功能 + 屏幕录制权限 | 首次使用时引导用户授权 |
| **多显示器 / HiDPI** | 低 | 嵌入窗口尺寸错位 | 通过 `getBoundingClientRect()` 精确同步 |
| **焦点竞争** | 低 | 嵌入窗口抢夺焦点 | 通过 `XSetInputFocus` / `SetFocus` 精确控制 |

## 十三、总结

本方案通过 **分层架构 + FFI 跨平台调用 + IPC 解耦** 实现了在 Nexus 面板中嵌入任意原生桌面程序的功能：

1. **架构清晰**：完全遵循现有面板系统的模式（类型 → 生命周期 → 渲染 → 快照）
2. **跨平台可行**：Linux/Windows 完全支持，macOS 通过镜像模式优雅降级
3. **最小侵入**：对现有代码的修改都是追加式，不改既有逻辑
4. **无需编译**：采用 FFI 方案，无需编译原生模块，Electron 升级无障碍
5. **风险可控**：关键风险都有缓解措施，实施分阶段推进

建议按 Phase 1 → 4 顺序推进，先保证 Linux 完整可用，再扩展到 Windows 和 macOS。
