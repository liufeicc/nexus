## 1. 功能概述

**浏览器面板**是 Nexus 的第三种面板类型（区别于终端面板和文件面板），专注于**网页浏览**。它与终端面板、文件面板共存于同一布局系统中，让你在终端旁边直接浏览网页（如文档、API 参考、调试页面等），无需切换到外部浏览器。

### 核心价值

| 价值点 | 描述 |
|--------|------|
| 减少上下文切换 | 不需要在终端和外部浏览器之间来回切换 |
| 多标签浏览 | 同时打开多个浏览器面板，对比查看不同页面 |
| 分屏浏览 | 与终端/文件面板并存，边看文档边写代码 |

### 使用场景

| 场景 | 说明 |
|------|------|
| 查阅文档 | 在浏览器面板中打开 API 文档，旁边终端执行命令 |
| 调试页面 | 在浏览器面板中打开本地开发服务器页面 |
| 多页面对比 | 左右分屏两个浏览器面板，对比不同页面效果 |
| 快速搜索 | 在浏览器面板中使用搜索引擎查找问题 |

## 3. 功能操作详解

### 3.1 创建浏览器面板

浏览器面板拥有独立的创建按钮，与文件面板和终端面板并列。

**操作步骤**:

```
方法一：通过工具栏按钮
1. 点击顶部工具栏的 [+ 浏览器] 按钮
2. 系统立即创建浏览器面板，默认导航到配置的默认网址
3. 如未配置默认网址，则显示空白页

方法二：右键菜单分屏
1. 在任意面板空白处右键点击
2. 选择"水平分屏"或"垂直分屏"
3. 在二级菜单中选择"浏览器面板"
4. 系统创建浏览器面板并添加到对应布局位置
```

### 3.2 网页导航

#### 3.2.1 地址栏导航

```
1. 点击地址栏输入框
2. 输入网址（如 "https://www.google.com"）或搜索关键词
3. 按 Enter 键确认
4. 页面开始加载，导航栏显示加载状态
```

**URL 自动补全规则**:
- 输入不带协议的地址（如 `www.google.com`）→ 自动添加 `https://`
- 输入包含 `.` 的地址（如 `example.com`）→ 视为网址
- 输入不包含 `.` 的文本（如 `react hooks`）→ 当作搜索关键词，使用 Google 搜索

#### 3.2.2 后退/前进

```
1. 点击导航栏的 ← 按钮返回上一页
2. 点击导航栏的 → 按钮前进到下一页
3. 或使用键盘快捷键 Alt+← / Alt+→
```

#### 3.2.3 刷新/停止

```
1. 点击 ↻ 按钮刷新当前页面
2. 页面加载时，↻ 变为 ■ 停止按钮
3. 点击 ■ 按钮停止加载
```

#### 3.2.4 加载进度

页面加载时，导航栏底部会显示一条蓝色的加载进度条，表示页面正在加载。

### 3.3 设置默认网址

默认网址是新建浏览器面板时自动导航到的地址。

```
1. 在地址栏中输入目标网址并导航
2. 点击地址栏右侧的 ⚙（齿轮）按钮
3. 系统提示"已设置为默认网址"
4. 下次新建浏览器面板时，自动导航到此地址
```

**清除默认网址**: 目前不支持通过 UI 清除。如需恢复空白页，可通过数据库直接删除 `browserDefaultUrl` 配置项。

### 3.4 URL 内联补全

地址栏支持内联补全功能，根据浏览历史自动提示最匹配的 URL。

#### 3.4.1 内联补全显示

```
1. 在地址栏中输入文字（如 "http"）
2. 输入框右侧出现灰色补全文字（如 "https://www.google.com" 的剩余部分）
3. 补全文字来自浏览历史中最匹配的记录
```

#### 3.4.2 接受补全

| 操作 | 效果 |
|------|------|
| `Tab` | 接受内联补全，将完整 URL 填入输入框 |
| `→` | 同上，接受补全 |

#### 3.4.3 切换补全建议

当有多条匹配的历史记录时：

| 操作 | 效果 |
|------|------|
| `↓` | 切换到下一条匹配的历史记录 |
| `↑` | 切换到上一条匹配的历史记录 |

#### 3.4.4 清除补全

| 操作 | 效果 |
|------|------|
| `Escape` | 清除补全和所有建议 |
| 继续输入 | 根据新的输入重新匹配 |
| 失去焦点 | 自动清除补全 |

### 3.5 浏览历史

浏览器面板会自动记录浏览历史，用于 URL 内联补全。

**历史记录规则**:

| 规则 | 说明 |
|------|------|
| 记录内容 | 每次导航成功后记录 URL 和页面标题 |
| 去重策略 | 相同 URL 不重复记录，仅更新访问时间 |
| 容量上限 | 最多保留 500 条记录 |
| 清理策略 | 超过 500 条时，自动删除最老的记录 |
| 排除项 | `about:blank` 不记录 |

### 3.6 与终端面板共存布局

浏览器面板可以像终端面板和文件面板一样参与分屏、拖拽交换等布局操作。

```
场景一：浏览器面板与终端面板左右分屏
┌────────────────────┬────────────────────┐
│  🌐 React Docs      │  $ npm run dev    │
│  https://react.dev  │  > Nexus dev      │
│  [网页内容]         │  $                │
│                    │                   │
│  浏览器面板         │  终端面板          │
└────────────────────┴────────────────────┘

场景二：左侧浏览器 + 右侧上下分屏终端
┌────────────────────┬────────────────────┐
│  🌐 API 文档        │  $ curl localhost │
│  [网页内容]         ├────────────────────┤
│                    │  $ git status     │
│                    │  终端面板 2        │
│  浏览器面板         │                   │
└────────────────────┴────────────────────┘
```

**布局操作支持**:

| 操作 | 说明 |
|------|------|
| 水平分屏 | 浏览器面板可以按水平方向分割 |
| 垂直分屏 | 浏览器面板可以按垂直方向分割 |
| 拖拽交换 | 浏览器面板可以与任何面板交换位置 |
| 分割线调整 | 浏览器面板与相邻面板间的分割线可拖拽调整大小 |
| 嵌套分割 | 浏览器面板可以作为嵌套布局的一部分 |

### 3.7 快照持久化

浏览器面板的状态会自动保存和恢复。

**自动保存的时机**:

| 操作 | 保存内容 |
|------|---------|
| 创建浏览器面板 | URL、面板标题、布局位置 |
| 导航到新页面 | 当前 URL、页面标题 |
| 调整面板大小 | 面板在布局中的比例 |
| 关闭面板 | 移除快照中的面板节点 |

**恢复内容**:

```
关闭会话后重新打开时，浏览器面板自动恢复：
- ✅ 当前 URL
- ✅ 页面标题
- ✅ 面板在布局中的位置
- ✅ 面板大小比例
```

## 4. 快捷键说明

### 4.1 浏览器面板快捷键

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Enter` | 导航到输入的地址 | 地址栏获得焦点时生效 |
| `Tab` | 接受内联补全 | 有补全建议时生效 |
| `→` | 接受内联补全 | 有补全建议时生效 |
| `↑` | 切换到上一条补全建议 | 有多条匹配时生效 |
| `↓` | 切换到下一条补全建议 | 有多条匹配时生效 |
| `Escape` | 清除补全和建议 | 地址栏获得焦点时生效 |
| `Alt+←` | 后退 | 返回上一页 |
| `Alt+→` | 前进 | 前进到下一页 |
| `Ctrl+R` / `Cmd+R` | 刷新 | 重新加载当前页面 |
| `Ctrl+L` / `Cmd+L` | 聚焦地址栏 | 快速定位到地址栏 |

### 4.2 与全局快捷键的关系

| 全局快捷键 | 浏览器面板中的行为 |
|-----------|-------------------|
| `Ctrl+T` | 新建面板 |
| `Ctrl+W` | 关闭当前浏览器面板 |
| `Ctrl+Tab` | 在多个面板间切换焦点 |

**注意事项**:
- 全局键盘快捷键（如 Ctrl+C/V）在地址栏中会被拦截
- Ctrl+C/V 在地址栏中正常工作（复制/粘贴文本）
- 快捷键仅在浏览器面板获得焦点时生效

## 5. 常见问题解答

### 5.1 入门问题

#### Q1: 浏览器面板和外部浏览器有什么区别？

| 对比项 | 外部浏览器 | 浏览器面板 |
|--------|-----------|-----------|
| 窗口 | 独立系统窗口 | 嵌入在 Nexus 面板中 |
| 标签页 | 多标签支持 | 每个面板一个页面 |
| 功能 | 完整浏览器功能 | 基础浏览 + 导航 |
| 用途 | 通用浏览 | 开发辅助、文档查阅 |

#### Q2: 新建浏览器面板时页面是空白的？

新面板会导航到配置的"默认网址"。如果未配置默认网址，则显示空白页。解决方法：
1. 在地址栏输入网址并导航
2. 点击 ⚙ 按钮设置为默认网址
3. 下次新建面板时自动加载此地址

### 5.2 导航问题

#### Q3: 输入地址后页面没反应？

- 检查地址格式是否正确（建议包含 `https://`）
- 不带协议的地址（如 `google.com`）会自动添加 `https://`
- 输入不包含 `.` 的文本会当作搜索关键词

#### Q4: 内联补全为什么不显示？

内联补全需要浏览历史数据。以下情况不会显示：
- 浏览历史为空（尚未导航过任何网址）
- 输入的文本与历史记录不匹配
- 历史记录已被清理（超过 500 条后自动清理旧记录）

### 5.3 性能问题

#### Q5: 多个浏览器面板会不会占用大量内存？

每个浏览器面板对应一个独立的 `WebContentsView` 实例，占用一定的内存和 GPU 资源。建议同时打开的浏览器面板不超过 4 个。不需要的面板及时关闭。

### 5.4 权限与异常

#### Q6: 某些页面无法加载？

- 检查网络连接是否正常
- 检查目标网站是否可访问
- 某些网站可能阻止了 Electron 的 User-Agent

#### Q7: 页面内容显示不正常？

浏览器面板使用 Electron 内置的 Chromium 引擎，版本可能与系统浏览器不同。如果遇到兼容性问题，建议：
- 在外部浏览器中打开该页面进行对比
- 确认不是网站本身的响应式布局问题

## 6. 附录

### A. 术语表

| 术语 | 说明 |
|------|------|
| WebContentsView | Electron 提供的原生 OS 层渲染组件，用于在面板中显示网页内容 |
| 内联补全 | 在输入框内显示的灰色补全文本，提示最匹配的 URL |
| 浏览历史 | 记录导航成功后的 URL 和页面标题，用于内联补全 |
| 默认网址 | 新建浏览器面板时自动导航到的地址，通过 ⚙ 按钮设置 |
| 快照 | 面板状态的持久化记录，用于会话恢复 |

### B. 技术说明

#### B.1 WebContentsView 渲染机制

浏览器面板使用 Electron 的 `WebContentsView` 进行网页渲染。`WebContentsView` 是原生 OS 层组件，渲染优先级高于 HTML DOM。这意味着：
- HTML 元素（如下拉框、模态框）无法覆盖 `WebContentsView`
- 地址栏的内联补全采用覆盖层方案，避免了被遮蔽的问题
- 右键菜单打开时，会自动截图并隐藏 `WebContentsView`，防止遮挡菜单

#### B.2 浏览历史存储

浏览历史存储在 SQLite 数据库的 `browser_history` 表中：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER | 主键 |
| `url` | TEXT | 网址（唯一） |
| `title` | TEXT | 页面标题 |
| `visited_at` | TEXT | 最后访问时间（ISO 8601 格式） |

**去重规则**: 相同 URL 的记录只保留一条，重复访问时更新 `visited_at` 字段。

### C. 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 地址栏输入 `about:blank` | 不记录到浏览历史 |
| 导航失败（网络错误等） | 地址栏保持当前值，不更新 |
| 面板关闭时正在加载 | 取消加载，不保存未完成的状态 |
| 多个面板同时导航到同一 URL | 各自独立记录，互不影响 |

## 设计决策记录

### ADR-001: 浏览器面板不依赖 PTY

**日期**: 2026-04-14

**决策**: 浏览器面板拥有独立的生命周期，不依赖 PTY 进程。

**采用理由**:
1. 网页浏览不需要 PTY，减少资源占用
2. 关闭浏览器面板时不需要确认，操作更轻量
3. 快照恢复时无需重建 PTY，速度更快

### ADR-002: URL 内联补全替代下拉框

**日期**: 2026-04-14

**决策**: 使用内联补全（灰色 ghost 文本）替代下拉列表。

**采用理由**:
1. `WebContentsView` 是原生 OS 层，HTML z-index 无法覆盖
2. 下拉框会被页面遮蔽，用户体验差
3. 内联补全无需覆盖 `WebContentsView`，直接在输入框内显示
4. 内联补全更简洁，符合现代浏览器地址栏交互习惯

### ADR-003: 默认网址配置在地址栏

**日期**: 2026-04-14

**决策**: 将"设置默认网址"功能以齿轮图标放在地址栏内，而非设置面板中。

**采用理由**:
1. 地址栏是最自然的入口，用户当前就在此操作 URL
2. 减少操作步骤，无需打开设置面板
