## 1. 项目概述

### 1.1 项目名称
**Nexus** - Terminal Views

### 1.2 项目定位
Nexus 是一款基于 Electron 开发的**图形化终端复用工具**，旨在提供类似 tmux 的多路复用功能，但通过直观的 GUI 界面降低使用门槛。

### 1.3 目标用户
- 觉得 tmux 快捷键复杂难记的开发者
- 需要频繁管理多个终端会话的前端/全栈工程师
- 希望可视化操作终端面板的 DevOps 人员
- 对 terminal workflow 有需求但不愿学习曲线陡峭工具的初级开发者

### 1.4 核心价值主张
| 对比维度 | tmux | Nexus |
|---------|------|--------|
| 学习成本 | 高（需记忆大量快捷键） | 低（图形化操作） |
| 操作方式 | 键盘快捷键 | 鼠标点击 + 可选快捷键 |
| 可视化程度 | 纯文本状态栏 | 完整 GUI 控制面板 |
| 配置方式 | 文本配置文件 | GUI 配置界面 |
| 跨平台一致性 | 依赖终端模拟器 | Electron 保证一致性 |

### 2.2 技术选型

#### 2.2.2 核心依赖库
| 功能 | 技术选型 | 说明 |
|------|---------|------|
| 终端渲染 | **xterm.js** | VS Code 同款，功能完善，性能优秀 |
| PTY 处理 | **node-pty** | Microsoft 出品，跨平台伪终端支持 |
| 状态管理 | **Zustand** | 轻量级，比 Redux 更简洁 |

#### 2.2.3 数据存储
| 数据类型 | 存储方案 | 说明 |
|---------|---------|------|
| 用户配置 | **better-sqlite3** | SQLite 数据库存储配置、主题、快捷键等 |
| 会话数据 | **better-sqlite3** | SQLite 数据库存储会话和快照信息 |
| 临时数据 | Memory + IPC | 运行时状态通过 Main Process 管理 |

**技术选型说明**：
- **better-sqlite3**：同步 API，适合 Electron 主进程，性能优异，单文件存储
- **数据库文件**：`%APPDATA%/Nexus/db.sqlite3` (Windows) / `~/.config/Nexus/db.sqlite3` (macOS/Linux)

### 2.2.4 数据库设计

**数据库文件位置**：
- Windows: `%APPDATA%/Nexus/db.sqlite3`
- macOS: `~/Library/Application Support/Nexus/db.sqlite3`
- Linux: `~/.config/Nexus/db.sqlite3`

**核心表结构**：

**1. configs 表（配置表）**
存储用户配置、主题设置、快捷键绑定、常用路径等：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| id | INTEGER PRIMARY KEY | 自增主键 |
| `key` | TEXT UNIQUE NOT NULL | 配置键名（唯一） |
| `value` | TEXT NOT NULL | 配置值（JSON 字符串） |
| updated_at | INTEGER | 更新时间戳 |

常用配置项：
- `theme`：当前主题配置（JSON）
- `shortcuts`：快捷键绑定（JSON 对象）
- `commonPaths`：常用路径列表（JSON 数组）
- `windowState`：窗口位置和大小（JSON）
- `terminalConfig`：终端默认配置（JSON）

**2. sessions 表（会话表）**
存储会话基本信息：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| id | INTEGER PRIMARY KEY | 自增主键 |
| name | TEXT NOT NULL | 会话名称 |
| created_at | INTEGER NOT NULL | 创建时间戳 |
| updated_at | INTEGER | 最后更新时间戳 |
| is_active | INTEGER DEFAULT 0 | 是否为当前激活会话（0/1） |

**3. session_snapshots 表（会话快照表）**
存储会话布局、面板信息等序列化数据：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| id | INTEGER PRIMARY KEY | 自增主键 |
| session_id | INTEGER NOT NULL | 关联的会话 ID（外键） |
| name | TEXT | 快照名称（可选） |
| layout_data | TEXT NOT NULL | 布局树数据（JSON 字符串） |
| active_panel_id | TEXT | 当前激活的面板 ID |
| panel_states | TEXT NOT NULL | 面板状态列表（JSON 字符串） |
| saved_at | INTEGER NOT NULL | 保存时间戳 |

**panel_states JSON 结构**：
```json
[
  {
    "panel_id": "pane-123",
    "pty_id": "pty-456",
    "title": "Terminal 1",
    "cwd": "/home/user/project",
    "shell_type": "bash",
    "size": { "width": 50, "height": 30 },
    "position": { "x": 0, "y": 0 }
  }
]
```

**数据库文件管理**：
```typescript
// 主进程中数据库初始化
import Database from 'better-sqlite3';
import { app } from 'electron';

const dbPath = path.join(app.getPath('userData'), 'db.sqlite3');
const db = new Database(dbPath);

// 自动创建表（若不存在）
db.exec(`
  CREATE TABLE IF NOT EXISTS configs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    key TEXT UNIQUE NOT NULL,
    value TEXT NOT NULL,
    updated_at INTEGER DEFAULT (strftime('%s', 'now'))
  );

  CREATE TABLE IF NOT EXISTS sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    updated_at INTEGER DEFAULT (strftime('%s', 'now')),
    is_active INTEGER DEFAULT 0
  );

  CREATE TABLE IF NOT EXISTS session_snapshots (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER NOT NULL,
    name TEXT,
    layout_data TEXT NOT NULL,
    active_panel_id TEXT,
    panel_states TEXT NOT NULL,
    saved_at INTEGER NOT NULL,
    FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
  );
`);
```

### 3.1 会话管理模块 (Session Manager)

#### 3.1.1 功能描述

interface Pane {
  id: string;

### 3.2 窗口/面板管理模块 (Pane Manager)

#### 3.2.1 功能描述
管理终端面板的分割、合并、调整大小和布局切换。

**核心理念**:
- 一个会话可以包含多个面板（一对多模式）
- 支持布局模式选择：左右分屏 / 上下分屏
- 新增面板时按照选定的布局模式自动排列

#### 3.2.2 树形嵌套布局（v2.0）

**设计目标**: 支持嵌套分屏，允许在不同面板上独立应用不同的分割方向。

**数据结构**:
```typescript
interface LayoutNode {
  type: 'horizontal' | 'vertical';  // 容器类型
  children: LayoutChild[];
}

interface PanelNode {
  type: 'panel';
  id: string;  // 面板 ID
}

type LayoutChild = LayoutNode | PanelNode;

interface Session {
  id: string;
  name: string;
  layout: LayoutNode;  // 布局树
  activePanelId: string;  // 当前选中的面板
  panelIds: string[];  // 所有面板 ID 列表（向后兼容）
}
```

**布局树示例**:
```
场景：面板 1 | [面板 2 / 面板 3]  (左右分屏 + 右侧上下分屏)

布局树:
{
  type: 'horizontal',
  children: [
    { type: 'panel', id: 'panel1' },
    {
      type: 'vertical',
      children: [
        { type: 'panel', id: 'panel2' },
        { type: 'panel', id: 'panel3' }
      ]
    }
  ]
}

视觉效果:
┌──────────────┬─────────────┐
│              │   面板 2     │
│   面板 1      ├─────────────┤
│              │   面板 3     │
└──────────────┴─────────────┘
```

**核心操作**:
| 操作 | 说明 | 实现函数 |
|------|------|---------|
| 分割面板 | 在指定面板上按方向分割 | `splitPanel(panelId, direction)` |
| 关闭面板 | 移除面板并简化空容器 | `closePanelFromTree(panelId)` |
| 渲染布局 | 递归渲染布局树为 DOM | `renderLayoutTree(node, container)` |

**自动简化规则**:
- 当容器只剩一个子节点时，自动展开该节点
- 关闭嵌套面板后，父容器自动简化

#### 4.2.3 新增面板流程

**场景一：当前会话没有面板（关闭所有面板后重新添加）**

1. 用户点击顶部工具栏"新增面板"按钮
2. 弹出路径选择对话框
3. 用户选择或输入起始路径
4. 创建新面板并绑定到当前会话
5. 面板占满整个容器区域

**场景二：当前会话已有 1 个面板，新增第 2 个面板**

1. 用户点击"新增面板"按钮
2. 根据当前布局模式（左右/上下）创建新面板
3. 两个面板平均分配空间

**场景三：当前会话已有多个面板，继续新增**

1. 用户点击"新增面板"按钮
2. 根据布局模式，所有面板重新平均分配空间

### 3.4 面板拖动交换（v2.0）

#### 3.4.1 功能描述

支持通过拖拽方式交换两个面板的位置，提供直观的面板布局调整方式。

**核心规则**:
- 按住面板标题栏不放，拖动到另一个面板位置
- 释放鼠标后，两个面板交换位置
- 只有一个面板时，不可拖动

**视觉效果**:
- 拖动中的面板：半透明虚化效果，跟随鼠标移动
- 目标面板：高亮边框提示

#### 3.4.2 实现要点

```javascript
// 拖动状态变量
let draggedPanel = null;      // 正在拖动的面板
let dragOverPanel = null;     // 当前拖入的面板
let dragGhost = null;         // 拖动时的虚影元素

// 核心函数
function swapPanels(panel1, panel2) {
    // 交换布局树中的节点位置
    swapNodesInTree(activeSession.layout, panel1Id, panel2Id);
    // 简化树结构
    activeSession.layout = simplifyTree(activeSession.layout);
    // 重新渲染布局
    renderLayoutTree(activeSession.layout, panelsContainer, panelRefs);
}
```

**拖动流程**:
1. `mousedown` - 创建虚影元素，原面板半透明
2. `mousemove` - 虚影跟随鼠标移动，检测目标面板
3. `mouseup` - 执行交换，清理虚影

**跨子树交换**:
- 同层级交换：直接交换数组位置
- 跨层级交换：递归查找并替换节点
- 跨子树交换：分别提取节点后双向替换

## 7. 技术难点与解决方案

### 7.1 难点一：多 PTY 进程同步管理

**问题描述**:
每个终端面板对应一个独立的 PTY 进程，需要确保：
- 输入输出正确路由到对应面板
- 进程生命周期与 UI 状态同步
- 内存泄漏防护

**解决方案**:
```typescript
// 主进程中维护 PTY 映射表
class PtyService {
  private ptyMap = new Map<string, IPty>();
  private sessionPtyMap = new Map<string, Set<string>>();

  createPty(sessionId: string, options: PtyOptions): string {
    const pty = spawn(options.shell, [], {
      name: options.name,
      cols: options.cols,
      rows: options.rows,
      cwd: options.cwd,
    });

    const ptyId = generateId();
    this.ptyMap.set(ptyId, pty);

    // 绑定数据转发
    pty.onData((data) => {
      this.mainWindow.webContents.send('pty-data', { ptyId, data });
    });

    return ptyId;
  }
}
```

### 7.2 难点二：动态布局状态同步

**问题描述**:
- 面板分割/合并操作需要实时更新布局状态
- 拖拽调整大小时需要平滑渲染
- 多窗口状态一致性

**解决方案**:
```typescript
// 使用 Zustand 管理布局状态

  panels: Panel[];
  splitPanel: (panelId: string, direction: 'h' | 'v') => void;

  removePanel: (panelId: string) => void;
}

// 使用 react-resizable-panels 处理拖拽
import { Panel, PanelGroup, PanelResizeHandle } from 'react-resizable-panels';

<PanelGroup autoSaveId="tviews-layout">

  </Panel>
  <PanelResizeHandle />
  <Panel defaultSize={70}>

```

### 7.3 难点三：会话恢复的上下文保持

**问题描述**:
- 如何恢复关闭的终端会话（历史命令、当前目录）
- 断线重连后的状态同步

**解决方案**:
```typescript
// 方案 A: 使用 shell 会话恢复功能（如 tmux/screen 作为后端）
// 方案 B: 保存工作目录，恢复时重新执行 cd 命令
// 方案 C: 使用终端历史记录保存

interface SessionSnapshot {
  panes: Array<{
    cwd: string;          // 工作目录
    commandHistory: string[]; // 命令历史（可选）
    shellType: string;    // Shell 类型
  }>;
}

// 恢复时
restoreSession(snapshot: SessionSnapshot) {
  snapshot.panes.forEach(pane => {
    const pty = this.ptyService.createPty({
      cwd: pane.cwd,
      // 发送 cd 命令切换到原目录
      onReady: () => pty.write(`cd "${pane.cwd}"\n`)
    });
  });
}
```

## 附录 D：SQLite 数据库设计方案

### D.1 设计概述

Nexus 采用 SQLite 作为统一的数据存储方案，使用 **better-sqlite3** 库作为 Node.js 绑定。SQLite 具有以下优势：
- **单文件存储**：便于备份和迁移
- **零配置**：无需额外数据库服务
- **ACID 兼容**：保证数据一致性
- **同步 API**：在 Electron 主进程中使用更直观

### D.2 数据库初始化

```typescript
import Database from 'better-sqlite3';
import { app } from 'electron';
import path from 'path';

const dbPath = path.join(app.getPath('userData'), 'db.sqlite3');
const db = new Database(dbPath);

// 创建表
db.exec(`
  CREATE TABLE IF NOT EXISTS configs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    key TEXT UNIQUE NOT NULL,
    value TEXT NOT NULL,
    updated_at INTEGER DEFAULT (strftime('%s', 'now'))
  );

  CREATE TABLE IF NOT EXISTS sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    updated_at INTEGER DEFAULT (strftime('%s', 'now')),
    is_active INTEGER DEFAULT 0
  );

  CREATE TABLE IF NOT EXISTS session_snapshots (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER NOT NULL,
    name TEXT,
    layout_data TEXT NOT NULL,
    active_panel_id TEXT,
    panel_states TEXT NOT NULL,
    saved_at INTEGER NOT NULL,
    FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
  );
`);
```

### D.3 数据迁移说明

从 v2.0 到 v2.1 的迁移策略：
- 现有的 JSON 配置文件将被读取并导入到 configs 表
- 旧的会话快照 JSON 文件将被导入到 session_snapshots 表
- 对应的 JSON 文件在导入后可被删除

## 附录 E：树形嵌套布局设计方案

### D.1 设计背景

当前 v1.0 版本采用扁平布局结构，所有面板共享同一个布局模式（左右/上下分屏），无法实现嵌套分屏。v2.0 引入树形布局结构，支持更灵活的分屏操作。

### D.2 核心概念

**布局树（Layout Tree）**：
- 根节点：表示整个面板容器的布局方向
- 容器节点：可包含多个子节点，定义子节点的排列方向
- 面板节点：叶子节点，代表单个终端面板

**分割操作**：
- 选中目标面板
- 选择分割方向（水平/垂直）
- 创建新容器，将原面板和新面板放入容器

**简化规则**：
- 当容器只剩一个子节点时，自动用子节点替换容器
- 避免产生冗余的容器节点

### D.3 数据结构

```typescript
// 布局树节点
interface LayoutNode {
  type: 'horizontal' | 'vertical';
  children: LayoutChild[];
}

// 面板节点
interface PanelNode {
  type: 'panel';
  id: string;
}

type LayoutChild = LayoutNode | PanelNode;

// 会话结构
interface Session {
  id: string;
  name: string;
  layout: LayoutNode;           // 布局树（v2.0 新增）
  activePanelId: string;        // 当前选中的面板
  panelIds: string[];           // 所有面板 ID（向后兼容）
}
```

### D.4 核心函数

| 函数 | 说明 | 参数 |
|------|------|------|
| `renderLayoutTree(node, container)` | 递归渲染布局树 | node: 布局树节点，container: 目标容器 |
| `splitPanel(targetPanelId, direction)` | 分割指定面板 | targetPanelId: 目标面板 ID，direction: 'horizontal'\|'vertical' |
| `closePanelFromTree(panelId)` | 关闭并移除面板 | panelId: 要关闭的面板 ID |
| `removeNodeFromTree(node, targetId)` | 递归删除节点并简化 | node: 根节点，targetId: 目标面板 ID |
| `replaceNodeInTree(node, targetId, newNode)` | 替换树中节点 | node: 根节点，targetId: 目标 ID，newNode: 新节点 |

### D.5 实现示例

**场景**：面板 1 | [面板 2 / 面板 3]

```javascript
// 初始布局（单个面板）
session.layout = {
  type: 'horizontal',
  children: [{ type: 'panel', id: 'panel1' }]
};

// 分割面板 1，创建面板 2（左右分屏）
session.layout = {
  type: 'horizontal',
  children: [
    { type: 'panel', id: 'panel1' },
    { type: 'panel', id: 'panel2' }
  ]
};

// 分割面板 2，创建面板 3（上下分屏）
session.layout = {
  type: 'horizontal',
  children: [
    { type: 'panel', id: 'panel1' },
    {
      type: 'vertical',
      children: [
        { type: 'panel', id: 'panel2' },
        { type: 'panel', id: 'panel3' }
      ]
    }
  ]
};
```

## 附录 F：AI 智能终端功能设计

### F.1 设计概述

Nexus 通过接入云端 LLM（如通义千问、OpenAI），实现 AI 自主判断和协助用户在终端上工作的能力。核心功能包括：

| 功能 | 说明 |
|------|------|
| **等待输入检测** | 终端等待用户输入时弹出提醒（shell 提示符、password prompt 等） |
| **工作流衔接** | 面板 A 工作完成后，面板 B 自动开始接下来的工作 |
| **错误监督** | 监督终端输出，有错误时提醒或关闭程序 |
| **自然语言工作流** | 用户用自然语言描述工作流程，AI 解析后自动编排 |

### F.2 技术架构要点

- **输出采集**: 主进程静默超时触发（2 秒无输出），环形缓冲区保留最近 500 行
- **LLM 调用**: 主进程执行（API Key 安全），最近 30 行上下文作为输入
- **工作流引擎**: 主进程状态机驱动，渲染进程崩溃不影响执行
- **API Key 存储**: Electron safeStorage 加密
- **默认授权**: "始终询问"，安全优先

### F.3 迭代计划

| 阶段 | 内容 | 预计周期 |
|------|------|---------|
| Phase 1 (MVP) | AI 基础监控：等待输入检测 + 错误检测 | 2 周 |
| Phase 2 | 工作流引擎：多面板自动衔接 + 确认对话框 | 3 周 |
| Phase 3 | 自然语言工作流：LLM 解析 + 面板角色绑定 | 2 周 |
| Phase 4 | 体验优化：误判反馈、通知历史、本地 LLM | 2 周 |
