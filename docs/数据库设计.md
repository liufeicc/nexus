## 1. 数据库概述

### 1.1 技术选型

| 项目 | 选型 |
|------|------|
| 数据库 | SQLite |
| Node.js 绑定 | better-sqlite3 |
| 数据库文件 | `~/.config/Nexus/db.sqlite3` (Linux) |

### 1.2 选型理由

- **单文件存储**: 便于备份和迁移
- **零配置**: 无需额外数据库服务
- **ACID 兼容**: 保证数据一致性
- **同步 API**: 在 Electron 主进程中使用更直观

### 1.3 数据库文件位置

| 平台 | 路径 |
|------|------|
| Windows | `%APPDATA%/Nexus/db.sqlite3` |
| macOS | `~/Library/Application Support/Nexus/db.sqlite3` |
| Linux | `~/.config/Nexus/db.sqlite3` |

### 2.2 configs 表 (配置表)

存储用户配置、主题设置、快捷键绑定、常用路径等。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增主键 |
| key | TEXT | UNIQUE NOT NULL | 配置键名 (唯一) |
| value | TEXT | NOT NULL | 配置值 (JSON 字符串) |
| updated_at | INTEGER | DEFAULT (strftime('%s', 'now')) | 更新时间戳 |

**常用配置项**:

| 配置键 | 值类型 | 说明 |
|-------|-------|------|
| `theme` | JSON | 当前主题配置 |
| `commonPaths` | JSON 数组 | 常用路径列表 |
| `windowState` | JSON | 窗口位置和大小 |
| `terminalConfig` | JSON | 终端默认配置 |
| `sidebarWidth` | NUMBER | 侧边栏宽度 (px) |
| `sidebarCollapsed` | BOOLEAN | 侧边栏收起状态 |
| `recentExpanded` | BOOLEAN | 最近会话列表展开状态 |
| `browserDefaultUrl` | STRING | 浏览器默认 URL |
| `agentEnabled` | BOOLEAN | 是否启用智能体 |
| `agentConfig` | JSON | 智能体 API 配置 (provider, apiUrl, apiKey, model, maxIterations, timeout, maxRetries, contextLength) |
| `agentSystemPrompt` | STRING | 智能体自定义系统提示词 |
| `agentSummaryModel` | JSON | 辅助模型配置 (用于上下文压缩摘要，结构同 agentConfig) |
| `agentInteractive` | BOOLEAN | 智能体危险命令交互式审批开关，默认 true |
| `llmContext` | STRING | 固定的 LLM 上下文提示（注入到 system prompt） |
| `webSearch` | JSON | 网络搜索工具配置 (provider, apiUrl, apiKey) |

**注意**: 快捷键配置（`shortcuts`）仅保存在内存中，不持久化到数据库。

### 2.3 sessions 表 (会话表)

存储会话基本信息。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增主键 |
| name | TEXT | NOT NULL | 会话名称 |
| created_at | INTEGER | NOT NULL | 创建时间戳 |
| updated_at | INTEGER | DEFAULT (strftime('%s', 'now')) | 最后更新时间戳 |
| is_active | INTEGER | DEFAULT 0 | 是否为当前激活会话 (0/1) |
| last_used_at | INTEGER | DEFAULT (strftime('%s', 'now')) | 最后使用时间戳 (用于最近使用列表) |

**索引**:
- `idx_sessions_is_active`: 加速活动会话查询
- `idx_sessions_last_used`: 加速最近使用会话查询

### 2.4 session_snapshots 表 (会话快照表)

存储会话布局、面板信息等序列化数据。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增主键 |
| session_id | INTEGER | NOT NULL | 关联的会话 ID (外键) |
| name | TEXT | | 快照名称 (可选) |
| layout_data | TEXT | NOT NULL | 布局树数据 (JSON 字符串) |
| active_panel_id | TEXT | | 当前激活的面板 ID |
| panel_states | TEXT | NOT NULL | 面板状态列表 (JSON 字符串) |
| saved_at | INTEGER | NOT NULL | 保存时间戳 |

**外键约束**:
```sql
FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
```

### 2.5 message_queue 表 (消息队列表)

存储动态岛通知消息，支持异步显示和状态管理。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增主键 |
| topic | TEXT | NOT NULL DEFAULT 'dynamic_island' | 消息主题 |
| title | TEXT | NOT NULL | 消息标题 |
| message | TEXT | NOT NULL DEFAULT '' | 消息内容 |
| type | TEXT | NOT NULL DEFAULT 'info' | 消息类型 (info/warning/error/success) |
| duration | INTEGER | | 显示时长（毫秒），为空则使用默认值 |
| icon | TEXT | | 图标标识 |
| status | TEXT | NOT NULL DEFAULT 'pending' | 状态 (pending/showing/dismissed) |
| created_at | INTEGER | NOT NULL | 创建时间戳 |
| updated_at | INTEGER | DEFAULT (strftime('%s', 'now')) | 更新时间戳 |

**索引**:
- `idx_mq_status_created`: 加速按状态和创建时间查询

### 2.6 memory_entries 表 (记忆条目表)

等价于 MEMORY.md / USER.md，存储智能体的持久化记忆笔记。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | TEXT | PRIMARY KEY | UUID 主键 |
| nexus_session_id | INTEGER | NOT NULL | 关联的 Nexus 会话 ID（外键） |
| scope | TEXT | NOT NULL DEFAULT 'memory' | 作用域 (`memory` / `user`) |
| content | TEXT | NOT NULL | 记忆内容 |
| created_at | INTEGER | DEFAULT (strftime('%s', 'now')) | 创建时间戳 |
| updated_at | INTEGER | DEFAULT (strftime('%s', 'now')) | 更新时间戳 |

**索引**:
- `idx_memory_session`: 加速按会话查询
- `idx_memory_scope`: 加速按会话+作用域查询

**外键约束**:
```sql
FOREIGN KEY (nexus_session_id) REFERENCES sessions(id) ON DELETE CASCADE
```

**作用域说明**:

| scope | 说明 | 字符限制 |
|-------|------|---------|
| `memory` | 智能体笔记（文件操作记录、项目上下文等） | 2200 字符 |
| `user` | 用户档案（偏好、习惯、知识背景等） | 1375 字符 |

### 2.7 memory_facts 表 (记忆事实表 + FTS5 全文检索)

存储智能记忆事实，支持 FTS5 全文搜索。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增主键（内部使用） |
| uuid | TEXT | NOT NULL UNIQUE | UUID 唯一标识（对外接口） |
| nexus_session_id | INTEGER | NOT NULL | 关联的 Nexus 会话 ID（外键） |
| content | TEXT | NOT NULL | 事实内容 |
| source | TEXT | DEFAULT 'agent' | 来源 (`agent` / `user` / `system`) |
| scope | TEXT | NOT NULL DEFAULT 'memory' | 作用域 (`memory` / `user`) |
| trust_score | REAL | DEFAULT 1.0 | 信任评分 (0.0 ~ 1.0) |
| retrieval_count | INTEGER | DEFAULT 0 | 被检索次数 |
| created_at | INTEGER | DEFAULT (strftime('%s', 'now')) | 创建时间戳 |
| updated_at | INTEGER | DEFAULT (strftime('%s', 'now')) | 更新时间戳 |

**索引**:
- `idx_facts_session`: 加速按会话查询
- `idx_facts_uuid`: 加速按 UUID 查询
- `idx_facts_trust_score`: 加速按信任评分排序
- `idx_facts_retrieval_count`: 加速按检索次数排序
- `idx_facts_created_at`: 加速按创建时间排序

**外键约束**:
```sql
FOREIGN KEY (nexus_session_id) REFERENCES sessions(id) ON DELETE CASCADE
```

### 2.8 memory_facts_fts (FTS5 全文搜索虚拟表)

SQLite FTS5 虚拟表，自动与 `memory_facts` 同步。

| 字段名 | 说明 |
|-------|------|
| rowid | 关联 memory_facts.rowid |
| content | 搜索内容（与 memory_facts.content 同步） |

**触发器**:

| 触发器 | 说明 |
|--------|------|
| `memory_facts_ai` | INSERT 后自动同步到 FTS |
| `memory_facts_ad` | DELETE 后自动从 FTS 删除 |
| `memory_facts_au` | UPDATE 后自动更新 FTS 索引 |

**分词器**: `unicode61`（支持中文分词）

**FTS5 内部表**: 创建 FTS5 虚拟表时，SQLite 会自动生成以下内部表，由 FTS5 引擎自行维护，**不应手动操作**：

| 表名 | 用途 |
|------|------|
| `memory_facts_fts` | FTS5 主虚拟表，用户搜索入口（`MATCH` 查询） |
| `memory_facts_fts_content` | 存储被索引的文本内容副本 |
| `memory_facts_fts_data` | 倒排索引数据（词 → 文档位置映射） |
| `memory_facts_fts_docsize` | 记录每个文档的词数，用于相关性排名 |
| `memory_facts_fts_idx` | 索引元数据和前缀树结构 |
| `memory_facts_fts_config` | FTS5 配置信息（分词器选项等） |

> 用户只对 `memory_facts_fts` 执行搜索查询，其余表由 SQLite 自动管理。数据通过触发器从 `memory_facts` 表同步到这些内部表。

### 2.9 memory_entities 表 (实体表)

从记忆内容中自动提取的实体（专有名词、项目名、路径等）。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增主键 |
| nexus_session_id | INTEGER | NOT NULL | 关联的 Nexus 会话 ID（外键） |
| name | TEXT | NOT NULL | 实体名称 |
| type | TEXT | NOT NULL | 实体类型 (file/project/person/path 等) |
| created_at | INTEGER | DEFAULT (strftime('%s', 'now')) | 创建时间戳 |

**唯一约束**: `UNIQUE(nexus_session_id, name, type)`

**索引**:
- `idx_entities_session`: 加速按会话查询

**外键约束**:
```sql
FOREIGN KEY (nexus_session_id) REFERENCES sessions(id) ON DELETE CASCADE
```

### 2.10 memory_entity_relations 表 (实体关联表)

存储实体之间的关系。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增主键 |
| entity_a_id | INTEGER | NOT NULL | 源实体 ID（外键） |
| relation | TEXT | NOT NULL | 关系类型 (depends_on/contains/related_to 等) |
| entity_b_id | INTEGER | NOT NULL | 目标实体 ID（外键） |
| fact_id | TEXT | | 关联的事实 UUID（外键，SET NULL） |
| created_at | INTEGER | DEFAULT (strftime('%s', 'now')) | 创建时间戳 |

**索引**:
- `idx_relations_entity_a`: 加速按源实体查询
- `idx_relations_entity_b`: 加速按目标实体查询
- `idx_relations_fact`: 加速按事实查询

**外键约束**:
```sql
FOREIGN KEY (entity_a_id) REFERENCES memory_entities(id) ON DELETE CASCADE,
FOREIGN KEY (entity_b_id) REFERENCES memory_entities(id) ON DELETE CASCADE,
FOREIGN KEY (fact_id) REFERENCES memory_facts(id) ON DELETE SET NULL
```

## 4. DAO 接口设计

### 4.1 ConfigDAO - 配置数据访问层

```typescript
class ConfigDAO {
  private db: Database;

  constructor(db: Database) {
    this.db = db;
  }

  /**
   * 保存配置项
   * @param key 配置键名
   * @param value 配置值 (自动序列化为 JSON)
   */
  save(key: string, value: any): void;

  /**
   * 获取配置项
   * @param key 配置键名
   * @returns 配置值对象，不存在返回 null
   */
  get(key: string): any | null;

  /**
   * 获取所有配置
   * @returns 配置对象 { [key]: value }
   */
  getAll(): { [key: string]: any };

  /**
   * 删除配置项
   * @param key 配置键名
   */
  delete(key: string): void;
}
```

### 4.2 SessionDAO - 会话数据访问层

```typescript
interface Session {
  id: number;
  name: string;
  createdAt: number;
  updatedAt: number;
  isActive: boolean;
}

class SessionDAO {
  private db: Database;

  constructor(db: Database) {
    this.db = db;
  }

  /**
   * 创建新会话
   * @param name 会话名称
   * @returns 创建的会话对象
   */
  create(name: string): Session;

  /**
   * 获取所有会话
   * @returns 会话列表
   */
  list(): Session[];

  /**
   * 获取单个会话
   * @param id 会话 ID
   * @returns 会话对象，不存在返回 null
   */
  getById(id: number): Session | null;

  /**
   * 更新会话名称
   * @param id 会话 ID
   * @param name 新名称
   */
  updateName(id: number, name: string): void;

  /**
   * 设置激活会话
   * @param id 会话 ID
   */
  setActive(id: number): void;
```

## 5. 数据库初始化

### 5.1 初始化脚本

```typescript
import Database from 'better-sqlite3';
import { app } from 'electron';
import path from 'path';
import fs from 'fs';

/**
 * 初始化数据库
 * @returns Database 实例
 */
export function initDatabase(): Database {
  // 确保数据目录存在
  const userDataPath = app.getPath('userData');
  const dbPath = path.join(userDataPath, 'db.sqlite3');

  // 确保目录存在
  if (!fs.existsSync(userDataPath)) {
    fs.mkdirSync(userDataPath, { recursive: true });
  }

  const db = new Database(dbPath);

  // 启用外键约束
  db.pragma('foreign_keys = ON');

  // 创建表
  db.exec(`
    -- 配置表
    CREATE TABLE IF NOT EXISTS configs (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      key TEXT UNIQUE NOT NULL,
      value TEXT NOT NULL,
      updated_at INTEGER DEFAULT (strftime('%s', 'now'))
    );

    -- 会话表
    CREATE TABLE IF NOT EXISTS sessions (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      name TEXT NOT NULL,
      created_at INTEGER NOT NULL,
      updated_at INTEGER DEFAULT (strftime('%s', 'now')),
      is_active INTEGER DEFAULT 0,
      last_used_at INTEGER DEFAULT (strftime('%s', 'now'))
    );

    -- 会话快照表
    CREATE TABLE IF NOT EXISTS session_snapshots (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      session_id INTEGER NOT NULL,
      name TEXT,
      layout_data TEXT NOT NULL,
      active_panel_id TEXT,
      panel_states TEXT NOT NULL,
      saved_at INTEGER NOT NULL,
      FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
    );

    -- 创建索引
    CREATE INDEX IF NOT EXISTS idx_sessions_is_active ON sessions(is_active);
    CREATE INDEX IF NOT EXISTS idx_snapshots_session_id ON session_snapshots(session_id);
    CREATE INDEX IF NOT EXISTS idx_snapshots_saved_at ON session_snapshots(saved_at);
  `);

  return db;
}
```

## 7. 数据迁移说明

### 7.1 从 JSON 文件迁移

如果系统中已存在 JSON 格式的配置文件和会话快照，可以通过以下脚本迁移到 SQLite:

```typescript
/**
 * 从 JSON 文件迁移数据到 SQLite
 */
export function migrateFromJson(db: Database, jsonDir: string) {
  // 迁移配置文件
  const configPath = path.join(jsonDir, 'config.json');
  if (fs.existsSync(configPath)) {
    const config = JSON.parse(fs.readFileSync(configPath, 'utf-8'));
    const stmt = db.prepare('INSERT OR REPLACE INTO configs (key, value) VALUES (?, ?)');

    for (const [key, value] of Object.entries(config)) {
      stmt.run(key, JSON.stringify(value));
    }
  }

  // 迁移会话快照
  const snapshotsPath = path.join(jsonDir, 'snapshots');
  if (fs.existsSync(snapshotsPath)) {
    const files = fs.readdirSync(snapshotsPath);
    const stmt = db.prepare(`
      INSERT INTO session_snapshots
      (session_id, name, layout_data, active_panel_id, panel_states, saved_at)
      VALUES (?, ?, ?, ?, ?, ?)
    `);

    for (const file of files) {
      const snapshot = JSON.parse(fs.readFileSync(path.join(snapshotsPath, file), 'utf-8'));
      stmt.run(
        snapshot.sessionId,
        snapshot.name,
        JSON.stringify(snapshot.layoutData),
        snapshot.activePanelId,
        JSON.stringify(snapshot.panelStates),
        snapshot.savedAt
      );
    }
  }
}
```
