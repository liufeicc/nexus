## 1. 界面操作与数据库操作完整映射

本节详细说明用户在界面上的每个操作如何对应到数据库操作。

---

### 1.1 应用启动流程

| 界面操作 | 触发时机 | 数据库操作 | 说明 |
|---------|---------|-----------|------|
| 应用启动 | 打开 Nexus | `ConfigDAO.getAll()` | 读取所有配置（主题、侧边栏状态、窗口状态等） |
| 应用启动 | 打开 Nexus | `SessionDAO.getActive()` | 获取上次活动的会话 |
| 应用启动 | 打开 Nexus | `SessionSnapshotDAO.getLatestBySession(sessionId)` | 获取活动会话的最新快照，恢复布局 |

**应用启动流程**:
```
1. 读取所有配置 → ConfigDAO.getAll()
   → 应用主题、恢复侧边栏状态
   → 恢复窗口位置和大小 (从 windowState 配置项)
   → 加载默认快捷键配置（内存中，不读数据库）

2. 获取活动会话 → SessionDAO.getActive()
   → 如果没有活动会话，创建默认会话

3. 获取会话快照 → SessionSnapshotDAO.getLatestBySession(sessionId)
   → 解析 layout_data 和 panel_states
   → 恢复面板布局和终端状态
```

### 1.2 应用关闭流程

| 界面操作 | 触发时机 | 数据库操作 | 说明 |
|---------|---------|-----------|------|
| 关闭窗口 | 关闭 Nexus | `ConfigDAO.save('windowState', state)` | 保存当前窗口位置和大小 |

**应用关闭流程**:
```
1. 用户点击窗口关闭按钮

2. 保存窗口状态
   ConfigDAO.save('windowState', { x, y, width, height, maximized })
   → INSERT OR REPLACE INTO configs (key, value, updated_at)
     VALUES ('windowState', '{"x":100,"y":100,...}', strftime('%s', 'now'))

3. 应用关闭（内存操作）
   - 终止所有 PTY 进程
   - 关闭数据库连接
   - 退出 Electron 应用
```

### 1.3 主题切换操作

| 界面操作 | 触发时机 | 数据库操作 | 说明 |
|---------|---------|-----------|------|
| 点击主题选项 | 用户选择新主题 | `ConfigDAO.save('theme', themeConfig)` | 保存主题配置 |
| 主题下拉菜单打开 | 点击主题按钮 | `ConfigDAO.get('theme')` | 读取当前主题，高亮对应选项 |

**主题切换流程**:
```
1. 用户点击主题下拉菜单中的"深蓝"选项

2. 更新主题配置
   ConfigDAO.save('theme', { name: 'deepblue' })
   → INSERT OR REPLACE INTO configs (key, value, updated_at)
     VALUES ('theme', '{"name":"deepblue"}', strftime('%s', 'now'))

3. 应用新主题（内存操作，不涉及数据库）
   - 根据主题名称查找内置配色方案
   - 更新 CSS 变量
   - 更新终端配色

4. 下次启动时自动加载该主题
```

1. 用户拖拽分割线调整侧边栏宽度

3. 下次打开应用时自动恢复该宽度

### 1.5 会话管理操作

| 界面操作 | 触发时机 | 数据库操作 | 说明 |
|---------|---------|-----------|------|
| 点击"新建会话"按钮 | 创建新会话 | `SessionDAO.create('new-shell')` | 创建新会话记录 |
| 点击"新建会话"按钮 | 创建新会话后 | `SessionDAO.setActive(newSessionId)` | 设置新会话为活动状态 |
| 点击会话项 | 切换会话 | `SessionDAO.setActive(sessionId)` | 设置选中会话为活动状态 (同时更新 last_used_at) |
| 点击会话项 | 切换会话后 | `SessionSnapshotDAO.getLatestBySession(sessionId)` | 获取会话快照恢复布局 |
| 点击会话项 (最近使用) | 刷新列表 | `SessionDAO.getRecent(3)` | 获取最近使用的 3 个会话 |
| 点击编辑按钮 | 重命名会话 | `SessionDAO.updateName(id, newName)` | 更新会话名称 |
| 点击删除按钮 | 删除会话 | `SessionDAO.delete(id)` | 删除会话（级联删除快照） |
| 右键菜单 → 新建会话 | 创建新会话 | `SessionDAO.create(name)` | 同新建会话按钮 |
| 右键菜单 → 重命名会话 | 重命名会话 | `SessionDAO.updateName(id, newName)` | 同编辑按钮 |
| 右键菜单 → 删除会话 | 删除会话 | `SessionDAO.delete(id)` | 同删除按钮 |

**最近使用会话说明**:
- 显示最近 3 个使用过的会话（包含当前活动会话）
- 按 `last_used_at` 字段倒序排序
- 切换会话时自动更新 `last_used_at` 字段

**新建会话流程**:
```
1. 用户点击左侧边栏底部的"[+] 新建会话"按钮

2. 创建新会话
   SessionDAO.create('new-shell')
   → INSERT INTO sessions (name, created_at, is_active)
     VALUES ('new-shell', strftime('%s', 'now'), 0)
   → 返回 { id: 5, name: 'new-shell', ... }

3. 设置新会话为活动状态
   SessionDAO.setActive(5)
   → UPDATE sessions SET is_active = 0 WHERE is_active = 1
   → UPDATE sessions SET is_active = 1 WHERE id = 5

4. 保存初始快照（空布局或默认布局）
   SessionSnapshotDAO.save(5, {
     layoutData: { type: 'horizontal', children: [] },
     activePanelId: null,
     panelStates: []
   })

5. UI 刷新，显示新会话项（带绿色状态点）
```

**切换会话流程**:
```
1. 用户点击会话列表中的"new-shell"项

2. 取消当前活动会话
   UPDATE sessions SET is_active = 0 WHERE is_active = 1

3. 设置新活动会话
   UPDATE sessions SET is_active = 1 WHERE id = 2

4. 获取新会话的最新快照
   SessionSnapshotDAO.getLatestBySession(2)
   → SELECT * FROM session_snapshots
     WHERE session_id = 2 ORDER BY saved_at DESC LIMIT 1

5. 解析快照数据，恢复面板布局
```

**删除会话流程**:
```
1. 用户点击会话项右侧的"×"按钮

2. 弹出确认对话框（UI 操作）

3. 用户点击"确定"后
   SessionDAO.delete(2)
   → DELETE FROM session_snapshots WHERE session_id = 2 (级联)
   → DELETE FROM sessions WHERE id = 2

4. 如果删除的是活动会话，自动切换到另一个会话
   SessionDAO.list() → 获取剩余会话
   SessionDAO.setActive(nextSessionId) → 设置新的活动会话
```

### 1.6 面板操作

| 界面操作 | 触发时机 | 数据库操作 | 说明 |
|---------|---------|-----------|------|
| 点击"新增面板"按钮 | 创建新面板 | `SessionSnapshotDAO.save(sessionId, data)` | 保存新增面板后的布局 |
| 点击"关闭面板"按钮 | 关闭面板 | `SessionSnapshotDAO.save(sessionId, data)` | 保存关闭面板后的布局 |
| 点击分屏按钮 | 分割面板 | `SessionSnapshotDAO.save(sessionId, data)` | 保存分割后的布局 |
| 拖动面板交换位置 | 交换完成 | `SessionSnapshotDAO.save(sessionId, data)` | 保存交换后的布局 |
| 面板内执行命令 | 实时操作 | 不操作数据库 | 命令输出在内存中 |
| 面板工作目录变化 | cwd 变化 | `SessionSnapshotDAO.save(sessionId, data)` | 保存最新面板状态 |

**新增面板流程**:
```
1. 用户点击顶部工具栏的"+"按钮

2. 弹出路径选择对话框（UI 操作）

3. 用户选择路径后创建新面板（内存操作）
   - 创建新的 PTY 实例
   - 创建 xterm.js 终端
   - 更新布局状态（内存）

4. 保存新布局到快照
   SessionSnapshotDAO.save(sessionId, {
     layoutData: { type: 'horizontal', children: [...] },
     activePanelId: 'new-panel-id',
     panelStates: [
       { panel_id: 'panel-001', cwd: '~', shell_type: 'bash', ... },
       { panel_id: 'panel-002', cwd: '~/projects', shell_type: 'bash', ... }
     ]
   })
   → INSERT INTO session_snapshots (...) VALUES (...)
```

**关闭面板流程**:
```
1. 用户点击面板标题栏的"×"按钮

2. 弹出确认对话框（UI 操作）

3. 用户点击"确定"后
   - 终止 PTY 进程（内存操作）
   - 从布局树中移除面板节点（内存）

4. 保存新布局到快照
   SessionSnapshotDAO.save(sessionId, {
     layoutData: { type: 'horizontal', children: [...] },
     activePanelId: 'remaining-panel-id',
     panelStates: [
       { panel_id: 'panel-001', cwd: '~', ... }
     ]
   })
```

### 1.7 快捷键配置

**注意**: 快捷键配置仅保存在内存中，不持久化到数据库。用户每次启动应用时使用默认快捷键配置。

| 界面操作 | 触发时机 | 数据操作 | 说明 |
|---------|---------|-----------|------|
| 修改快捷键绑定 | 保存配置 | 内存操作 | 更新当前会话的快捷键映射（不保存） |
| 恢复默认快捷键 | 点击重置按钮 | 内存操作 | 恢复为默认快捷键配置 |

**快捷键配置流程**:
```
1. 应用启动时
   → 加载默认快捷键配置到内存
   → 默认配置示例:
     {
       "splitHorizontal": "Ctrl+Shift+H",
       "splitVertical": "Ctrl+Shift+V",
       "closePanel": "Ctrl+Shift+W",
       "newSession": "Ctrl+Shift+T"
     }

2. 用户在设置中修改快捷键
   → 更新内存中的快捷键映射
   → 应用新的快捷键绑定（内存操作）

3. 应用关闭
   → 快捷键配置不保存
   → 下次启动时恢复为默认配置
```

### 1.8 常用路径管理操作

| 界面操作 | 触发时机 | 数据库操作 | 说明 |
|---------|---------|-----------|------|
| 打开路径选择器 | 弹出对话框 | `ConfigDAO.get('commonPaths')` | 读取常用路径列表 |
| 选择自定义路径 | 确认新建会话/面板 | `ConfigDAO.save('commonPaths', paths)` | 保存新路径到常用列表 |
| 新建会话选择路径 | 确认路径 | `ConfigDAO.save('commonPaths', paths)` | 如果路径不存在则添加到列表开头 |

**常用路径管理说明**:
- 常用路径存储在 `configs` 表的 `commonPaths` 配置项中
- 格式为 JSON 数组：`[{ name: string, path: string, icon?: string }]`
- 最多保留 10 个常用路径
- 新路径添加到列表开头
- 已存在的路径不会重复添加

**新建会话时保存路径流程**:
```
1. 用户点击"新建会话"按钮

2. 弹出路径选择对话框
   ConfigDAO.get('commonPaths')
   → SELECT value FROM configs WHERE key = 'commonPaths'
   → 显示常用路径列表

3. 用户输入自定义路径并确认，例如：/home/liufei/my-project

4. 保存路径到常用位置
   - 检查路径是否已存在
   - 如果不存在，添加到列表开头
   - 最多保留 10 个，超出则移除最后一个

   ConfigDAO.save('commonPaths', [
     { name: 'my-project', path: '/home/liufei/my-project', icon: '📁' },
     ...其他路径
   ])
   → INSERT OR REPLACE INTO configs (key, value, updated_at)
     VALUES ('commonPaths', '[{...}]', strftime('%s', 'now'))

5. 创建会话和面板
```

## 3. 会话管理操作汇总表

| 操作 | DAO 方法 | 涉及表 | 说明 |
|-----|---------|-------|------|
| 创建会话 | `SessionDAO.create(name)` | sessions | 创建新会话记录 |

## 4. 完整业务流程示例

### 4.1 用户新建会话并添加面板

```
1. 创建会话
   SessionDAO.create('new-shell')
   → INSERT INTO sessions (name, created_at) VALUES ('new-shell', 1743321600)
   → 返回 { id: 1, name: 'new-shell', ... }

2. 设置新会话为活动状态
   SessionDAO.setActive(1)
   → UPDATE sessions SET is_active = 0
   → UPDATE sessions SET is_active = 1 WHERE id = 1

3. 保存会话快照 (包含布局和新面板信息)
   SessionSnapshotDAO.save(1, {
     layoutData: { type: 'horizontal', children: [...] },
     activePanelId: 'panel-001',
     panelStates: [{ panel_id: 'panel-001', cwd: '~', ... }]
   })
   → INSERT INTO session_snapshots (session_id, layout_data, active_panel_id, panel_states, saved_at)
       VALUES (1, '{"type":"horizontal"...}', 'panel-001', '[{...}]', 1743321600)
```

### 4.2 用户切换会话

```
1. 取消当前活动会话
   → UPDATE sessions SET is_active = 0 WHERE is_active = 1

2. 设置新活动会话
   → UPDATE sessions SET is_active = 1 WHERE id = 2

3. 获取新会话的最新快照 (用于恢复布局)
   SessionSnapshotDAO.getLatestBySession(2)
   → SELECT * FROM session_snapshots WHERE session_id = 2 ORDER BY saved_at DESC LIMIT 1
```

### 4.3 应用启动时恢复状态

```
1. 获取所有配置
   ConfigDAO.getAll()
   → SELECT key, value FROM configs
   → 恢复主题、快捷键、侧边栏状态等

2. 获取活动会话
   SessionDAO.getActive()
   → SELECT * FROM sessions WHERE is_active = 1

3. 获取活动会话的最新快照
   SessionSnapshotDAO.getLatestBySession(activeSessionId)
   → 恢复面板布局
```

### 4.4 用户删除会话

```
1. 删除会话 (外键级联删除快照)
   SessionDAO.delete(id)
   → DELETE FROM session_snapshots WHERE session_id = ? (级联)
   → DELETE FROM sessions WHERE id = ?
```

### 4.5 布局变化时自动保存

```
当用户分割面板/关闭面板/调整面板大小时：

SessionSnapshotDAO.save(sessionId, {
  layoutData: newLayout,
  activePanelId: activePanel,
  panelStates: panelStates
})
→ INSERT INTO session_snapshots (...) VALUES (...)
→ 每次布局变化都插入新记录，保留历史版本
```

## 5. 事务处理

涉及多个表的操作需要使用事务保证原子性:

```typescript
// 示例：创建会话并设置活动状态
db.transaction((name: string) => {
  // 1. 取消所有活动状态
  db.prepare('UPDATE sessions SET is_active = 0').run();

  // 2. 创建新会话
  const result = db.prepare(`
    INSERT INTO sessions (name, created_at, is_active)
    VALUES (?, strftime('%s', 'now'), 1)
  `).run(name);

  return { id: result.lastInsertRowid, name, isActive: true };
})(name);
```
